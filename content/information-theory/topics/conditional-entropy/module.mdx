# Conditional entropy

Conditional entropy H(Y|X) measures how much uncertainty remains about Y once the value of X is known. It is the information-theoretic version of "residual variance" or "prediction error" when trying to guess Y from X.

It plays a starring role in the definitions of mutual information, channel capacity, and rate-distortion functions.

## Definition of Conditional Entropy

<!-- section: definition -->

The conditional entropy of Y given X is the expected entropy of the conditional distributions:

$H(Y|X) = \sum_x p(x) H(Y|X=x) = -\sum_{x,y} p(x,y) \log_2 p(y|x).$

It can also be expressed as H(Y|X) = H(X,Y) - H(X).

**ELI5**

- After you look at X, on average how many more bits do you still need to describe Y?
- If X is the month and Y is the temperature in a city, H(Y|X) is the average remaining uncertainty about temperature once you know which month it is.

**Worked Example:**

X fair coin, Y = X (copy). Then given X you know Y perfectly, so H(Y|X=x)=0 for both values of x. Thus H(Y|X)=0.

## Interpretation and Properties

<!-- section: interpretation-properties -->

H(Y|X) ≥ 0 always.

H(Y|X) = H(Y) exactly when X and Y are independent.

H(Y|X) ≤ H(Y), with strict inequality unless I(X;Y)=0.

Conditioning on more variables can only decrease (or keep) the conditional entropy: H(Y|X,Z) ≤ H(Y|X).

**ELI5**

- Learning X can never make you more uncertain about Y on average; it can only help or do nothing.
- If X tells you nothing about Y they are independent and H(Y|X)=H(Y).

**Worked Example:**

X uniform on {1,2,3,4}, Y = X mod 2. Then H(Y)=1 bit (fair bit). But given X, Y is known, so H(Y|X)=0 < 1.

## Chain Rule and Relations

<!-- section: chain-rule-relations -->

The chain rule H(X,Y) = H(X) + H(Y|X) is the most useful identity.

It rearranges to H(Y|X) = H(X,Y) - H(X) and symmetrically H(X|Y) = H(X,Y) - H(Y).

Also, H(X,Y|Z) = H(X|Z) + H(Y|X,Z) (chain rule conditioned on Z).

**ELI5**

- Total joint uncertainty = uncertainty in X plus the extra uncertainty in Y once X is known.
- You can "pay" the uncertainty of X first, then pay only the conditional part for Y.

**Worked Example:**

Suppose H(X)=2, H(Y)=3, H(X,Y)=4. Then H(Y|X) = 4-2 = 2 bits, and H(X|Y) = 4-3 = 1 bit. X is more informative about Y than vice versa.

## Common Mistakes

- **Thinking H(Y|X) can be negative**: It cannot; conditional entropy is always ≥ 0.
- **Confusing H(Y|X) with I(X;Y)**: I(X;Y) = H(Y) - H(Y|X) is the reduction; H(Y|X) is what is left.
- **Writing H(Y|X) using only marginals p(y)**: You need the conditional p(y|x) or the joint.
- **Forgetting that H(Y|X) is an average**: For some particular x the conditional entropy H(Y|X=x) can be larger than H(Y), even though the average cannot.
- **Assuming H(Y|X)=0 implies X determines Y with probability 1 for every x**: It only requires that for almost every x (under p(x)) the conditional is deterministic.
- **Mixing conditional entropy with conditional probability**: They are related but one is a number (uncertainty), the other a function of the realized value.
