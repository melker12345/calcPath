# Entropy (information theory)

Entropy is the central quantity of information theory. It measures the average amount of information (or equivalently, uncertainty) produced by a stochastic source. Claude Shannon introduced it in 1948 as the unique (up to scaling) function satisfying a few intuitive axioms about how information should behave when sources are combined or observed.

Entropy sets the fundamental limits for both data compression and many other information-processing tasks.

## Definition of Entropy

<!-- section: definition -->

For a discrete random variable X taking values in a finite alphabet with probability mass function p(x), the Shannon entropy is

$H(X) = - \sum_{x} p(x) \log_2 p(x).$

When the sum is over an infinite support the series must converge. By convention 0 log 0 = 0.

Entropy can be interpreted as the average number of bits needed to describe the outcome of X using an optimal code.

**ELI5**

- Entropy is the "average surprise." Every time you see a realization of X you get on average H(X) bits of new information.
- If you already know the distribution, entropy tells you how hard it is to guess the next draw on average.

**Worked Example:**

1. A biased coin with $P(\text{heads})=0.9$, $P(\text{tails})=0.1$ has entropy
2. $H = -0.9 \log_2(0.9) - 0.1 \log_2(0.1) \approx 0.469$ bits.
3. You can guess "heads" and be right $90\%$ of the time, so on average far less than 1 bit of uncertainty per flip.

## Properties of Entropy

<!-- section: properties -->

Key properties:

- $H(X) \ge 0$, with equality iff X is deterministic.
- $H(X)$ is maximized by the uniform distribution: $H(X) \le \log_2 |\mathcal{X}|$.
- $H(f(X)) \le H(X)$ for any function f (data processing).
- Entropy is concave in the distribution p.
- For independent X,Y: $H(X,Y) = H(X) + H(Y)$.

The binary entropy function $h(p) = -p\log_2 p -(1-p)\log_2(1-p)$ appears everywhere in information theory.

**ELI5**

- You cannot create uncertainty by relabeling or processing the outcomes deterministically; you can only lose information.
- Independent coins add their entropies: two fair coins together have 2 bits.

**Worked Example:**

1. Consider two independent fair bits $X$ and $Y$. Then $H(X,Y) = H(X) + H(Y) = 2$ bits.
2. The joint distribution is uniform over 4 possibilities, confirming $\log_2(4) = 2$.

## Calculating Entropy

<!-- section: calculating -->

To compute H for a given pmf, simply plug into the formula. For small alphabets it is straightforward; for large ones one often uses the binary entropy or closed forms (e.g., geometric, Poisson in limit).

A useful table of small examples:

| Distribution          | Probabilities          | Entropy (bits) |
|-----------------------|------------------------|----------------|
| Deterministic         | (1)                    | 0              |
| Fair coin             | (0.5, 0.5)             | 1              |
| Biased coin (0.9)     | (0.9, 0.1)             | ≈0.469         |
| Fair 4-ary            | (0.25 each)            | 2              |
| Example in text       | (0.5,0.25,0.125,0.125) | 1.75           |

**ELI5**

- Make a table of probabilities, multiply each by -log2 of itself, and average (sum). That is the entropy.
- When symbols are very unequal, entropy drops because you can predict the common ones easily.

**Worked Example:**

1. Symbols $A,B,C,D$ with $p = 0.4, 0.3, 0.2, 0.1$.
2. $H = -0.4 \log_2(0.4) -0.3 \log_2(0.3) -0.2 \log_2(0.2) -0.1 \log_2(0.1)$.
3. $\approx 0.4 \cdot 1.322 + 0.3 \cdot 1.737 + 0.2 \cdot 2.322 + 0.1 \cdot 3.322 \approx 1.971$ bits.

## Common Mistakes

- **Using log base e or 10 without converting**: The numerical value changes; remember the conversion factor log₂e ≈ 1.4427 when going from nats.
- **Forgetting the negative sign**: -sum p log p is positive; dropping the minus gives a negative nonsense value.
- **Treating 0 log 0 as undefined in code**: In math we define lim p→0 p log p = 0; always add a guard in implementations.
- **Confusing entropy of the distribution with entropy of a sample**: H is a property of p, not of any particular draw or finite sequence.
- **Assuming H(X|Y) = H(X) - something always**: Only the mutual information term subtracts; conditional entropy can be computed directly.
- **Ignoring that entropy rate ≠ single-symbol entropy for processes with memory**: Real text, images, etc. have dependence that must be modeled for accurate rate.
