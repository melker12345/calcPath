# Algorithmic information theory

Algorithmic information theory (AIT) quantifies the information content of individual objects (strings, files, mathematical statements) rather than ensembles. The central notion is Kolmogorov complexity K(x): the length of the shortest program (in a fixed universal programming language) that outputs x and halts.

AIT provides a rigorous foundation for Occam's razor, induction, and the notion of randomness for single sequences. It is intimately connected with Shannon theory in the asymptotic regime but offers insights for finite objects that probability alone cannot.

## Kolmogorov Complexity (recap / intro)

<!-- section: kolmogorov-intro -->

K(x) is the minimal |p| such that a fixed prefix-free universal Turing machine U on input p outputs x: K(x) = min { |p| : U(p) = x }.

Because of the invariance theorem, K(x) defined with respect to any two universal machines differs by at most a constant (the size of a translator program).

K(x) is uncomputable (by reduction to the halting problem). We can only approximate it from above by running all short programs in dovetail fashion and taking the shortest one that has output x so far.

**ELI5**

- K(x) is the ultimate compressed size of x: the smallest description that lets a computer reconstruct x exactly.
- A random-looking string of length n usually has K(x) ≈ n; a string like 000...0 or the digits of π has much smaller K because there is a short program that generates it.

**Worked Example:**

1. The string consisting of one million zeros has $K \approx \log(\text{of the program 'print 0 one million times'})$ which is a few dozen bits, independent of the length.
2. A typical random million-bit string requires a program that essentially lists all the bits; $K \approx 1\,000\,000 + O(1)$.

## Algorithmic Probability and Solomonoff Induction

<!-- section: solomonoff -->

The algorithmic probability (Solomonoff prior) of a string x is

m(x) = ∑_{p : U(p)=x} 2^{-|p|}

the total probability mass of all programs that output x. It is a universal semimeasure: it dominates every computable probability distribution (up to a constant factor).

Solomonoff induction uses m to predict the next symbol: the predictive probability is m(x0)/m(x) for continuation 0, etc. It is the optimal predictor in the sense that its total expected squared error is bounded for any computable environment.

**ELI5**

- Instead of assuming a particular probability model, consider every possible program that could have generated the data you have seen, weighted by 2^{-length of program}. Simpler (shorter) programs that fit the data get higher prior weight.
- Future predictions are the weighted average over all consistent programs. This automatically embodies Occam's razor in a formal, prior-free way.

**Worked Example:**

1. After observing a long string of heads from a coin, the short programs 'always output heads' and 'fair coin' both have mass.
2. The universal mixture will predict the next toss is heads with probability $> 1/2$, but will gradually approach the true bias as more data arrives, because the 'always heads' program will be falsified if a tail ever appears.

## Incompressibility, Martin-Löf Randomness, and Chaitin's Omega

<!-- section: incompressibility-chaitin -->

A string x is (Martin-Löf) random if K(x) ≥ |x| - c for a fixed constant c (i.e., it is incompressible). Most strings are random; the number of strings of length n with K(x) < n - c is < 2^{n-c}.

Chaitin's Ω = ∑_{p halts} 2^{-|p|} is the halting probability of the universal machine. It is a random real (its binary digits are algorithmically random), uncomputable, and contains the answers to all halting problems in its bits.

AIT implies that 'truth' in mathematics is incompressible: for any consistent formal system there is a constant c such that the system cannot prove K(x) > |x| + c for any x.

**ELI5**

- A random string has no pattern that a program shorter than itself can exploit; any description is basically as long as the string.
- Ω is a single real number that 'knows' whether every possible program halts. You can compute as many digits as you want by running all programs up to that length, but you can never know all the digits.

**Worked Example:**

1. There are $2^n$ binary strings of length $n$. At most $2^{n-c}-1$ of them can be outputs of programs shorter than $n-c$.
2. Therefore at least a $(1-2^{-c})$ fraction of strings of length $n$ satisfy $K(x) \geq n-c$; they are incompressible / random.

## Common Mistakes

- **Confusing Kolmogorov complexity K(x) with Shannon entropy H(X)**: K is for an individual string; H is an average over a distribution. For a string drawn from p, K(x) ≈ -log p(x) + O(1) with high probability (the asymptotic equipartition property in AIT).
- **Thinking K(x) is computable**: It is not; if it were we could solve the halting problem.
- **Believing that 'random-looking' strings are always incompressible**: Some strings that pass statistical tests can still have short descriptions (e.g., the binary digits of a computable irrational that looks random).
- **Forgetting the additive constant when comparing K across different machines or languages**: K_{U}(x) and K_{V}(x) differ by at most the size of a program that translates between U and V.
- **Assuming that because Ω is random its digits are 'patternless' in every sense**: While algorithmically random, the digits of Ω are highly structured (they encode halting information) and are not 'typical' under any computable measure in a strong sense.
- **Using AIT directly for practical compression**: K is uncomputable; practical compressors are approximations that work well on the kinds of redundancy humans and computers actually produce.
