# Rate–distortion function

The rate-distortion function R(D) is the central object of lossy source coding. For a given source distribution and single-letter distortion measure, R(D) is the smallest number of bits per symbol that must be sent so that the receiver can reconstruct the source sequence with average distortion no larger than D.

It is both a lower bound (converse) that no code can beat and, by the rate-distortion theorem, an asymptotically achievable rate.

## The Rate-Distortion Function R(D)

<!-- section: the-rd-function -->

Formally,

$R(D) = \inf_{p(\hat{x}|x):\ \mathbb{E}[d(X,\hat{X})]\leq D} I(X;\hat{X})$

where the infimum is over all conditional pmfs (or pdfs) p(\hat x | x) such that the expected distortion is at most D. The infimum is achieved by some test channel p^*(\hat x | x) that is usually not the same as the forward channel that would be used in a communication problem.

R(D) is a non-increasing convex function of D, with R(0) = H(X) (or h(X) for continuous) and R(D) = 0 for all D ≥ D_max where D_max is the distortion of the best zero-rate reproduction (e.g. the value that minimizes E[d(X, c)] over constants c).

**ELI5**

- R(D) draws the curve 'how many bits do I really need if I'm allowed this much average error?'
- Because it is an infimum over all possible ways the reproduction can statistically depend on the source, it gives the absolute best possible compression performance.

**Worked Example:**

For any source, at D=0 the only allowed test channels have \hat X = X with probability 1, so I(X; \hat X) = H(X) and thus R(0) = H(X). This recovers the lossless source coding theorem as the special case of rate-distortion theory.

## Parametric Computation via the s-Parameter

<!-- section: parametric-s -->

It is often convenient to work with the Lagrangian formulation. For a slope parameter s ≤ 0 define

$R_s = \min_{p(\hat x|x)} [ I(X;\hat X) + s \mathbb{E} d(X,\hat X) ]$

Then the minimizing test channel for each s gives a point (D(s), R(s)) on the R(D) curve, where D(s) = - dR_s / ds (by envelope theorem). Varying s from 0 to -∞ traces the entire convex curve from (D_max, 0) to (0, H(X)).

This parametric form is the basis of the Blahut-Arimoto algorithm and of most analytic derivations.

**ELI5**

- Instead of fixing the distortion budget D and minimizing rate, we penalize distortion with a price s and minimize the combined 'cost' rate + s·distortion. The right price s gives exactly the operating point we want.
- Sweeping the price from 'distortion is free' (s=0) to 'distortion is infinitely expensive' (s→-∞) draws the whole R(D) frontier.

**Worked Example:**

For the Gaussian quadratic case the optimal test channel is always additive Gaussian noise whose variance is chosen so that total MSE = D. The parameter s turns out to be s = -1/(2 D ln 2) (in bits); plugging in recovers the familiar R(D) = ½ log(σ²/D).

## Closed-Form Examples and Bounds

<!-- section: closed-form-bounds -->

Only a handful of sources have known closed-form R(D):

- Bernoulli(p) with Hamming distortion: R(D) = h_2(p) - h_2(D) for D ≤ min(p,1-p) (more generally the binary R(D) formula).
- Gaussian with quadratic distortion: R(D) = ½ log(σ²/D) for D ≤ σ².
- Some other exponential-family sources with matching distortion measures.

Useful bounds include the Shannon lower bound R(D) ≥ h(X) - h(D) (for difference distortion measures), and various upper bounds obtained from specific test channels (e.g., the Gaussian test channel for non-Gaussian sources gives an upper bound on R(D)).

**ELI5**

- For the 'nicest' sources (Gaussian, binary) we have exact formulas that look like 'entropy minus the entropy of the allowed error'.
- For everything else we compute numerically or bound R(D) between a lower bound (often the Shannon lower bound) and the performance of a good practical scheme.

**Worked Example:**

Binary source with p=0.1 (entropy h_2(0.1)≈0.469). For Hamming D=0.05, R(D) = h_2(0.1) - h_2(0.05) ≈ 0.469 - 0.286 = 0.183 bits. This is strictly less than the entropy, as expected for lossy.

## Common Mistakes

- **Plotting or using R(D) for D larger than the support of the distortion measure**: R(D) is defined to be zero past the point where a constant reproduction already meets D.
- **Forgetting that R(D) depends on both the source distribution and the distortion measure**: Changing d(·,·) from squared error to absolute error or to a perceptual metric changes the function completely.
- **Confusing the test channel p(\hat x | x) with a communication channel**: The test channel is an artificial mathematical object used only to compute the mutual information; it need not be realizable as a physical channel.
- **Using the high-rate approximation R(D) ≈ h(X) - ½ log(2πe D) outside its validity region**: The approximation is good only when D is small relative to the source variance and the optimal reproduction is fine-grained.
- **Thinking that achieving R(D) is easy in practice**: The theorem is asymptotic in block length; for finite n one pays a penalty, and finding the optimal codebook or test channel is hard.
- **Overlooking that R(D) is an average per-symbol quantity**: For a long block the total rate is n R(D) + o(n); the o(n) term can matter for short blocks or when strong concentration is required.
