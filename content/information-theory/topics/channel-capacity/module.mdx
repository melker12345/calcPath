# Channel capacity

Channel capacity is the fundamental limit, discovered by Shannon, on how fast information can be transmitted reliably over a noisy communication channel. It is the highest rate (in bits per channel use or bits per second) at which one can send messages with the probability of error made arbitrarily small by using sufficiently long codes.

The noisy-channel coding theorem asserts that rates below capacity are achievable and rates above capacity are not.

## Definition via Mutual Information

<!-- section: definition -->

For a discrete memoryless channel with transition probabilities p(y|x), the capacity is

$C = \max_{p(x)} I(X;Y),$

where the maximum is over all possible input distributions p(x), and I(X;Y) is the mutual information between input and output.

The maximizing distribution p^*(x) is the capacity-achieving input distribution.

**ELI5**

- Capacity answers: "What is the fastest speed (in bits per transmission) I can talk over this noisy link so that the listener can still understand me perfectly if I use a good enough code?"
- It is the largest mutual information you can create between what you send and what arrives.

**Worked Example:**

A channel that simply copies the input (no noise) has I(X;Y)=H(X). The maximum is log| X |, achieved when the input is uniform. Hence C = log |alphabet| bits per use — you can send one symbol per transmission perfectly.

## The Binary Symmetric Channel (BSC)

<!-- section: binary-symmetric -->

The BSC flips each transmitted bit independently with probability p (the crossover probability). Its capacity is

$C = 1 - h_2(p),$

where h_2(p) = -p log₂ p - (1-p) log₂(1-p) is the binary entropy function.

The optimal input is the uniform (fair coin) distribution.

**ELI5**

- Even if the channel flips 10% of the bits, you can still send information at a positive rate (about 0.53 bits per transmission) by using error-correcting codes.
- When p=0.5 the channel is useless (pure noise) and C=0.

**Worked Example:**

p=0.11, h_2(0.11)≈0.5, therefore C≈0.5 bits per use. With a good code you can transmit at 0.5 bits per channel use with error probability as small as you like.

## Gaussian Channels and the Shannon Limit

<!-- section: gaussian -->

For the additive white Gaussian noise (AWGN) channel with average power constraint P and noise variance N (SNR = P/N), the capacity (in the appropriate normalization) is

$C = \frac{1}{2} \log_2 (1 + \text{SNR}).$

This is the famous formula that shows capacity grows only logarithmically with signal power.

**ELI5**

- Doubling power gives you less than 0.5 extra bits per transmission. To gain a full extra bit you need to roughly quadruple the power (6 dB).
- This is why engineers care so much about bandwidth and clever coding rather than just "turning up the volume."

**Worked Example:**

SNR = 7 (≈8.5 dB). Then C = 0.5 log2(8) = 1.5 bits per transmission. This is the ultimate speed limit for that power and noise level; no coding scheme can exceed it.

## Common Mistakes

- **Confusing capacity with the raw bit rate or symbol rate**: Capacity is the *reliable information* rate after optimal coding; you may send symbols faster but with higher error, or use bandwidth expansion.
- **Thinking capacity is achieved by sending at the highest power all the time**: The optimal input distribution is often not the "full power" or uniform; water-filling is needed for parallel or frequency-selective channels.
- **Forgetting that capacity is an asymptotic quantity**: It is achievable only in the limit of long block lengths; for short packets there is a penalty (finite-blocklength information theory).
- **Believing that any code operating below C works**: The theorem says there *exist* codes that achieve low error; finding them (or good approximations) is the job of coding theory.
- **Mixing capacity with rate-distortion**: Capacity is about reliable transmission (channel coding); rate-distortion is about lossy compression (source coding).
- **Assuming the capacity formula for Gaussian is the same in continuous time without bandwidth**: The classic C = B log2(1+SNR) formula includes bandwidth B; the discrete-time version above normalizes to one use per "dimension."
