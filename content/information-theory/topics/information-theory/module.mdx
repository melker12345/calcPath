# Information theory

Information theory is the mathematical study of the quantification, storage, and communication of information. Founded by Claude Shannon in his landmark 1948 paper "A Mathematical Theory of Communication," it provides the theoretical limits for data compression, reliable transmission over noisy channels, and many modern technologies from the internet to machine learning.

It separates the engineering problem of communication from the semantics of the messages being sent.

## What is Information?

<!-- section: what-is-information -->

Information is fundamentally about reducing uncertainty. If an outcome is certain (probability 1), learning it tells you nothing new. The rarer or more surprising an event, the more information it carries when observed.

Formally, information theory models sources as random variables or stochastic processes and measures information via probabilities.

**ELI5**

- Imagine a weather forecast: "It will be sunny" on a desert planet carries almost zero information because it is expected. "It will snow" would be highly informative.
- A fair coin flip resolves one bit of uncertainty; learning it happened "heads" cuts the possibilities exactly in half.

**Worked Example:**

A message arrives saying a specific person won the lottery where there were 1,000,000 equally likely tickets. The self-information of that specific outcome is I = -log₂(1/1 000 000) = 20 bits approximately. Most other messages (someone else won) would carry far less surprise on average.

## Quantifying Information

<!-- section: quantifying-information -->

The self-information (or surprisal) of a specific outcome x with probability p(x) is

$I(x) = -\log_2 p(x)$ bits.

The entropy of a discrete random variable X is the expected self-information:

$H(X) = -\sum_x p(x) \log_2 p(x) = \mathbb{E}[-\log_2 p(X)]$.

Entropy is always non-negative, maximized for the uniform distribution over a given support, and is measured in bits when using log base 2 (or nats with natural log).

**ELI5**

- Entropy answers: on average, how many fair yes/no questions do you need to identify the value of X?
- A perfectly fair coin has 1 bit of entropy. A coin that always lands heads has 0 bits of entropy—you already know the answer.

**Worked Example:**

Consider a random variable with four outcomes having probabilities 1/2, 1/4, 1/8, 1/8.

H = -(1/2 log₂(1/2) + 1/4 log₂(1/4) + 1/8 log₂(1/8) + 1/8 log₂(1/8))

= 0.5*1 + 0.25*2 + 0.125*3 + 0.125*3 = 0.5 + 0.5 + 0.375 + 0.375 = 1.75 bits.

## Applications and Scope

<!-- section: applications-scope -->

Information theory gives fundamental limits: the source coding theorem bounds lossless compression by H(X); the noisy-channel coding theorem bounds reliable transmission rate by channel capacity C. These are not engineering suggestions but mathematical facts that no scheme can beat in the asymptotic regime.

Modern applications include zip files, error-correcting codes in hard drives and wireless, deep learning (cross-entropy loss), and cryptography.

**ELI5**

- Think of entropy as the "irreducible size" of a message on average. You can get close to it with good codes but never sustainably below it.
- Channel capacity tells you the highest speed (in bits per transmission) at which you can talk over a noisy phone line with arbitrarily small error probability.

**Worked Example:**

A source produces independent symbols A (p=0.5), B (p=0.25), C (p=0.25). Its entropy is 1.5 bits per symbol. For a long sequence of 1000 symbols the typical total information content is about 1500 bits, so any lossless compressor must output at least roughly 1500 bits on average.

## Common Mistakes

- **Confusing information with meaning or importance**: A random string of digits can have high information content (entropy) yet be completely meaningless to a human.
- **Using the wrong logarithm base**: For bits always use log₂; using log₁₀ or ln changes the numerical value by a constant factor (log₂e ≈ 1.4427 for nats to bits).
- **Treating entropy as a property of a single string**: Entropy is defined for a distribution (ensemble); Kolmogorov complexity is the individual-string analogue.
- **Assuming maximum entropy is always uniform over the alphabet**: It is, but only when all symbols are possible; if the support size is smaller the max entropy is lower.
- **Mixing up H(X) and I(X;Y)**: Entropy H(X) is uncertainty in X alone; mutual information I(X;Y) is the reduction in that uncertainty when Y is observed.
- **Forgetting that real sources have memory**: The simple i.i.d. entropy formula is only the starting point; entropy rate accounts for dependence in sequences.
