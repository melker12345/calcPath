# Predicate Logic

Predicate logic, or first-order logic (FOL), extends propositional logic by allowing us to talk about objects and their properties and relations. Instead of treating "Socrates is mortal" as an atomic proposition, we decompose it as $M(s)$ where $s$ names Socrates and $M$ is the predicate "is mortal".

We also add quantifiers: "for all" ($\forall$) and "there exists" ($\exists$). This lets us express statements such as "every natural number has a successor" or "there exists a prime larger than 100". FOL is expressive enough for most of mathematics yet still decidable in important fragments and has a complete proof system (Gödel's completeness theorem).

## Predicates, Constants, Functions, and Variables

<!-- section: predicates -->

A **predicate** (or relation symbol) of arity $k$ takes $k$ arguments and yields a truth value. $P(x)$ is a unary predicate (property); $R(x,y)$ is binary (relation).

**Constants** (names) denote specific objects: $a, b, c, \ldots$ or $0, \pi, \text{Socrates}$.

**Function symbols** produce new objects from old: $f(x)$, $+(x,y)$, $\text{successor}(n)$.

**Variables** $x,y,z$ stand for arbitrary objects; they are not yet assigned meaning until we interpret or quantify.

An **atomic formula** is a predicate symbol applied to terms (constants, variables, or function applications), e.g. something like R applied to f(a) and x.

**ELI5**

- Predicates are like adjectives or verbs with blanks: "__ is even", "__ loves __". Fill the blanks with names or variables and you get a claim that is true or false of those particular things.
- Constants are proper names. Functions are "machines" that take input objects and output a new object (successor function turns 3 into 4).

**Worked Example:**

In the language of arithmetic: constant $0$, function $S$ (successor), predicate $=$ (equality, usually built-in).

Atomic formulas: $S(0) = S(S(0))$ is false under standard interpretation; $x = 0$ is atomic but open.

## Quantifiers and Their Meaning

<!-- section: quantifiers -->

The **universal quantifier** $\forall x \, \phi(x)$ means "for every object $x$ in the domain, $\phi(x)$ holds".

The **existential quantifier** $\exists x \, \phi(x)$ means "there is at least one object $x$ in the domain such that $\phi(x)$ holds".

Quantifiers range over the **domain** (universe) of the interpretation — the set of objects we are talking about. Changing the domain changes truth.

Quantifiers can nest: $\forall x \exists y \, (y > x)$ ("there is no largest number").

**ELI5**

- $\forall$ is "no matter which one you pick, the claim is still true". Like a universal warranty: every customer gets a refund.
- $\exists$ is "I can point to one that works". Like "there exists a winning move" — you don't need to list all, just show one example suffices.
- Order matters: $\forall x \exists y$ (for every x I can find a y) is weaker than $\exists y \forall x$ (there is one y that works for every x).

**Worked Example:**

"Every person has a mother": $\forall x \exists y \, \text{Mother}(y, x)$.

"There is someone who is everyone's mother": $\exists y \forall x \, \text{Mother}(y, x)$.

These are not equivalent; the second is much stronger (and false in normal human domains).

## Scope, Free and Bound Variables

<!-- section: scope -->

The **scope** of a quantifier is the subformula it governs, usually indicated by parentheses or convention.

A variable occurrence is **bound** if it lies inside the scope of a quantifier that uses the same variable name; otherwise it is **free**.

A formula with no free variables is a **sentence** (closed formula) and can be true or false in an interpretation. Open formulas (with free vars) express properties or relations.

Example: $\forall x (P(x) \to Q(x))$ is closed. $P(x) \to Q(y)$ has free $x$ and $y$.

Renaming bound variables is safe (alpha-equivalence): $\forall x \phi(x) \equiv \forall y \phi(y)$.

**ELI5**

- Free variables are like pronouns without antecedent in the sentence: "He is tall" — who? You need context (an assignment of values).
- Bound variables are dummy: "For every person x, if x is a student then x studies" — the 'x' is just a placeholder; you can rename it to 'z' without changing meaning.
- Scope is the "jurisdiction" of the quantifier — like parentheses in arithmetic. $\exists x (P(x) \land Q(x))$ vs $(\exists x P(x)) \land Q(x)$ — different meanings.

**Worked Example:**

In $\exists x (P(x) \land \forall x Q(x))$, the second $x$ is bound by the inner $\forall$, the first by the $\exists$. The formula is equivalent (after renaming) to $\exists x (P(x) \land \forall y Q(y))$.

## Interpretations and Satisfaction

<!-- section: interpretations -->

An **interpretation** (or structure, model) for a first-order language consists of:

- A non-empty domain $D$.
- For each constant $c$, an element $c^I \in D$.
- For each $k$-ary function $f$, a function $f^I : D^k \to D$.
- For each $k$-ary predicate $P$, a relation $P^I \subseteq D^k$.

An assignment of values to free variables, together with the interpretation, determines whether a formula is satisfied.

We write $\mathcal{M} \models \phi$ when sentence $\phi$ is true in model $\mathcal{M}$.

**ELI5**

- An interpretation is "picking the actual universe and what the symbols mean in it". The same sentence "$\forall x (P(x) \to Q(x))$" can be true in one universe (all P things are also Q) and false in another.
- Satisfaction is "this particular assignment of objects to the free slots makes the claim hold in this model".

**Worked Example:**

Language with unary $P$. Domain $D = \{1,2,3\}$. Let $P^I = \{1,2\}$.

Then $\exists x P(x)$ is true; $\forall x P(x)$ is false; $\exists x \neg P(x)$ is true.

If we change domain to $\{1,2\}$ and $P^I$ full, $\forall x P(x)$ becomes true.

## Common Mistakes

- **Quantifier scope and domain:** Treat quantifiers with care for scope and domain rather than like propositional connectives or English plurals. "All that glitters is not gold" is classically misparsed; the logical form is usually $\neg \forall x (G(x) \to \text{Gold}(x))$.

- **Quantifier order:** Swapping quantifier order without justification is invalid. $\forall x \exists y$ ("everyone has a mother") is not equivalent to $\exists y \forall x$ ("someone is mother of everyone").

- **Non-empty domains:** Forgetting that the domain must be non-empty leads to errors. Some texts allow empty domains; classical FOL usually requires non-empty.

- **Free versus bound:** Confusing free and bound variables is common. Writing proofs that treat a free $x$ as if universally quantified without the $\forall$ intro rule is incorrect.

- **Intended model versus validity:** Assuming that truth in the "intended" model implies validity (truth in all models) gets the direction wrong. Soundness goes the other way.

- **Nested quantifier capture:** Using the same variable name for nested quantifiers without renaming leads to accidental capture.
