# Ordinal number

Ordinal numbers are a generalization of the natural numbers that describe not only size but the *order type* of well-ordered sets. In set theory they are realized as special sets: transitive sets well-ordered by membership.

They provide the canonical "rulers" against which any well-ordering can be measured.

## Definition and Basic Properties

<!-- section: definition-ordinals -->

A set $\\alpha$ is an *ordinal* if:
1. $\\alpha$ is transitive: every element of an element is an element (if $\\beta \\in \\alpha$ and $\\gamma \\in \\beta$ then $\\gamma \\in \\alpha$).
2. $\\alpha$ is well-ordered by $\\in$: every nonempty subset has an $\\in$-least element.

We write $\\beta < \\alpha$ to mean $\\beta \\in \\alpha$, and $\\beta \\le \\alpha$ for $\\beta \\in \\alpha$ or $\\beta = \\alpha$.

**ELI5**

- An ordinal is like a "perfectly stacked tower of boxes" where each box contains exactly all the boxes below it, and there are no gaps or cycles.
- The natural numbers 0,1,2,3... are the first infinite "tower"; $\\omega$ is the whole tower as one object.

**Worked Example:**

0 = $\\emptyset$ (empty tower).
1 = $\{0\}$ (one box containing the empty tower).
2 = $\{0,1\}$ .
These are all ordinals. The set $\{0,2\}$ is not transitive (1 $\\in$ 2 but 1 not in the set) and is not an ordinal.

## Order Types

<!-- section: order-types -->

Every well-ordered set is order-isomorphic to a unique ordinal. That ordinal is called the *order type* of the well-ordering.

This gives a total order on ordinals themselves: for any two ordinals, one is an initial segment of the other (or they are equal).

**ELI5**

- Instead of saying "this line of people is ordered like the counting numbers," we say its order type is $\\omega$.
- Different orderings of the same size can have different order types (e.g., 1,2,3,... vs ...,3,2,1 have types $\\omega$ and $\\omega^*$ respectively).

**Worked Example:**

The order type of the usual ordering on $\\mathbb{N}$ is $\\omega$. The order type of the negative integers with usual order (...,-3,-2,-1) is the reverse, often written $\\omega^*$; it is not equal to any ordinal because ordinals are well-ordered "forward."

## Successor and Limit Ordinals

<!-- section: successor-limit -->

- The *successor* of $\\alpha$ is $\\alpha + 1 = \\alpha \\cup \\{\\alpha\\}$.
- A *limit ordinal* is one that is not a successor (it is the supremum of all smaller ordinals).

$\\omega$ is the smallest limit ordinal. Every ordinal is either 0, a successor, or a limit.

**ELI5**

- Successor ordinals are "right after" another one, like 5 is right after 4.
- Limit ordinals are "at the end of an infinite climb," like $\\omega$ is after all the finite numbers but has no last finite number immediately before it.

**Worked Example:**

$\\omega + 1 = \\omega \\cup \\{\\omega\\}$ is the order type of naturals followed by one extra point at the end. $\\omega + \\omega$ is two copies of $\\omega$ back to back; it is a limit ordinal (sup of $\\omega + n$ for finite n).

## Examples and Applications

<!-- section: examples-ordinals -->

- Finite ordinals = natural numbers.
- $\\omega, \\omega+1, \\omega+\\omega, \\omega^2, \\omega^\\omega, \\epsilon_0, \\dots$ (ordinal arithmetic).
- First uncountable ordinal $\\omega_1$.
- Ordinals are used to index transfinite recursions, well-founded induction, and the ranks of sets in the cumulative hierarchy.

**ELI5**

- You can keep counting past infinity: after all naturals comes $\\omega$, then $\\omega+1$, etc. Each new "number" is a new ordinal describing a longer well-ordering.
- In proofs, "transfinite induction up to $\\alpha$" lets you assume the statement for all smaller ordinals and prove it for $\\alpha$.

**Worked Example:**

To define the rank of every set by transfinite recursion: rank($\\emptyset$) = 0; rank($x$) = sup { rank(y) + 1 | y $\\in$ x }. This recursion is justified because $\\in$ is well-founded (regularity) and ordinals provide the "clock."

## Common Mistakes

- **Confusing ordinals with cardinals**: Every finite ordinal is a cardinal, but $\\omega + 1$ is an ordinal whose cardinality is still $\\aleph_0$; not every ordinal measures "size" in the cardinal sense.
- **Thinking all well-orderings of naturals have type $\\omega$**: The ordering 0,2,4,...1,3,5,... has type $\\omega + \\omega$.
- **Forgetting transitivity**: A set of ordinals that is not downward-closed (transitive) is not itself an ordinal.
- **Identifying $\\omega_1$ with the continuum**: $\\omega_1$ is the smallest uncountable ordinal; whether its cardinality equals $2^{\\aleph_0}$ is the continuum hypothesis.
- **Assuming ordinal arithmetic is commutative**: $\\omega + 1 \\neq 1 + \\omega$; order types care about "which comes first."
