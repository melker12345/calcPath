# Forcing (mathematics)

Forcing is a powerful technique invented by Paul Cohen in 1963 to prove independence results in set theory. It allows one to start from a model of ZFC and adjoin new "generic" sets in a controlled way, producing a larger model in which previously undecidable statements become true or false.

Cohen used forcing to show that the continuum hypothesis and the axiom of choice are independent of ZF (more precisely, that ¬CH and ¬AC are consistent with ZF, assuming ZF itself is consistent).

## What Forcing Is

<!-- section: what-is-forcing -->

Given a ground model M of (a fragment of) ZFC and a partial order (poset) P ∈ M that M thinks is a forcing notion, one constructs a *generic filter* G ⊆ P that is "generic over M" (intersects every dense subset of P that belongs to M).

One then builds the *generic extension* M[G] consisting of all sets that can be "named" by P-names in M and interpreted using G.

The extension M[G] is again a model of ZFC (if M was), and it contains new sets (the generic object decoded from G) while preserving a great deal of the structure of M.

**ELI5**

- Think of P as a collection of "partial descriptions" or "promises" about a new object you want to add (e.g., finite pieces of a new real number).
- A generic G is an infinite coherent collection of these promises that "answers every question" that the ground model M can ask (by meeting all dense sets M can define).
- M[G] is the universe you get by adding the object described by G and closing under the set-theoretic operations that M already knew.

**Worked Example:**

The simplest forcing: Cohen forcing to add one new real. Let P be the set of all finite functions from ω to 2 (finite binary sequences), ordered by extension. A generic filter G corresponds to an infinite binary sequence (the union of the conditions in G) that is not in the ground model M. In M[G] there is a new subset of ω (the Cohen real).

## Posets and Generic Filters

<!-- section: posets-and-generics -->

A poset P is a partial order (usually assumed separative or with a maximum element 1). Stronger conditions give more information.

A filter G on P is *M-generic* if for every dense D ⊆ P with D ∈ M we have G ∩ D ≠ ∅.

Density means that below any condition there is a stronger condition in D; genericity says G "decides" every question M can pose.

**ELI5**

- Conditions are like "finite approximations to the object being added."
- Dense sets are "requirements" or "questions" that M can express ("make sure the real is not equal to this particular old real", "make sure this ordinal is collapsed or preserved", etc.).
- Genericity guarantees every requirement is eventually satisfied.

**Worked Example:**

In Cohen forcing, the dense set D_r = {p ∈ P | p is incompatible with the characteristic function of a particular ground-model real r} forces the generic real to differ from r at some point. Genericity ensures the generic real differs from every ground-model real, hence is new.

## Using Forcing to Prove Independence

<!-- section: forcing-to-prove-independence -->

To show that a statement φ is not provable from ZFC (assuming consistency):
- Start with a model M ⊨ ZFC.
- Find a poset P ∈ M such that M thinks "there is a condition that forces ¬φ" (or that in M[G] we will have ¬φ).
- Show that the forcing preserves enough of ZFC (or at least the consistency strength) so that M[G] ⊨ ZFC + ¬φ.

Cohen's original argument for ¬CH: force with a poset that adds ℵ₂ many new reals (or a product/iteration that makes 2^ℵ₀ ≥ ℵ₂) while preserving cardinals (using ccc posets). In the extension the old ℵ₁ is still uncountable but there are at least ℵ₂ many reals.

For ¬AC one uses symmetric extensions or permutation models (or class forcing) in which a Dedekind-finite infinite set of reals or a countable family of pairs without a choice function appears.

**ELI5**

- You design a "recipe" (the poset) for adding exactly the kind of new object that will make the statement you want to be independent become true (or false).
- You prove that the recipe does not destroy the axioms of set theory and does not accidentally make the statement true in the old model.

**Worked Example:**

To make 2^ℵ₀ = ℵ₂ (or larger), use the finite-support product of ℵ₂ many copies of Cohen forcing (or the Lévy collapse, or random forcing, etc.). The ccc property ensures no cardinals are collapsed, so the old ℵ₁ remains ℵ₁ in the extension, but many new reals are added, pushing the continuum up.

## Preservation Theorems

<!-- section: preservation-theorems -->

A central part of forcing technology is proving that certain posets preserve cardinals, cofinalities, or other properties:
- ccc (countable chain condition) posets preserve all cardinals and cofinalities ≥ ℵ₁.
- Proper posets preserve ℵ₁ and stationary subsets of ω₁.
- More generally, one classifies posets by how much they destroy (or preserve) of the ground model.

The forcing relation p ⊩ φ ("p forces φ") is defined in the ground model by a recursive definition and is the key tool for proving what will be true in the extension without having the generic yet.

**ELI5**

- Not every "recipe" for adding new sets is harmless; some posets collapse cardinals or destroy stationary sets.
- Preservation theorems tell you which posets are safe for which purposes (e.g., "if you only care about not collapsing ℵ₁, you may use any proper poset").

**Worked Example:**

A poset is ccc if it has no uncountable antichain. Cohen forcing is ccc. By a delta-system argument, any uncountable set of conditions has two compatible ones, so no new countable sequence of ground-model ordinals can be added; hence ℵ₁ is preserved.

## Common Mistakes

- **Thinking forcing 'adds' sets directly to the true universe V**: Forcing is a relative consistency technique inside a model M; the extension M[G] is another model, not necessarily a subclass of the true V.
- **Confusing the poset with the generic object**: The poset P lives in the ground model and describes possible approximations; the generic G is the actual infinite coherent object (usually not in M).
- **Believing every forcing destroys cardinals**: Many important posets (ccc, proper, semi-proper) are carefully designed *not* to collapse cardinals or to collapse only specific ones.
- **Forgetting the definability of the forcing relation**: All the combinatorial work (density arguments, preservation) is done inside the ground model using the definable relation ⊩; you never need the generic to be present to reason about what it will force.
- **Assuming forcing is the only way to obtain independence**: There are other methods (inner models, large cardinals, symmetric extensions, etc.), but forcing is the most flexible for "making the continuum large" or "adding counterexamples to choice."
