# Inverse trigonometric functions

Inverse trig recover the angle from a known sine, cosine or tangent value. Their ranges are restricted to make them functions (one-to-one).

## Inverse Trigonometric Functions

<!-- section: inverse-trig -->

The inverse trig functions recover an angle from a known trig value within a restricted principal range where the function is one-to-one.

- $\arcsin x$ : domain [-1,1], range [-π/2, π/2]
- $\arccos x$ : domain [-1,1], range [0, π]
- $\arctan x$ : domain all reals, range (-π/2, π/2)
- $\text{arccsc}, \text{arcsec}, \text{arccot}$ defined similarly with their own conventions.

These are essential for solving trig equations ("what angle has this sine?") and in calculus for integrals and derivatives.

**ELI5**

- arcsin(x) is "the angle whose sine is x", but we pick the answer from a standard "principal" slice of the circle so the machine always gives one consistent answer.

**Worked Example:**

**Compute arctan(1) + arcsin(0.5)**

1. arctan(1) = π/4 (45°).
2. arcsin(0.5) = π/6 (30°).
3. Sum: π/4 + π/6 = 5π/12.

## Common Mistakes

- **Range of inverse functions**: arcsin range is [-π/2,π/2]; arccos [0,π]. Using the wrong branch or forgetting the range produces answers outside the principal value.
- **Calculator mode**: arccos etc return in current angle mode; check.
- **arcsin(sin x) ≠ x for all x**: Only true inside the principal range; outside it gives the co-terminal in range.
- **Domain restrictions**: arcsin and arccos only defined for [-1,1]; arctan all reals.
- **Confusing arctan with 1/tan**: arctan is inverse function, not reciprocal (which is cot).
- **Quadrant when solving**: After arcsin you may need to add π - result for other quadrant solutions in equations.
