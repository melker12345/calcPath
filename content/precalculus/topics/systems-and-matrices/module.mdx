# Systems & Matrices

Linear systems, matrix arithmetic, determinants, inverses, and Gaussian elimination.

## System of linear equations

<!-- section: linear-systems -->

Systems of linear equations arise everywhere: balancing chemical reactions, finding intersection of lines/planes, optimizing with constraints, and describing steady states in networks.

### Systems of Linear Equations

A **system of linear equations** (or linear system) is a collection of equations of the form a1 x1 + a2 x2 + … + an xn = b.

Solutions are values of the variables that satisfy every equation simultaneously.

A system can have:
- Exactly one solution (independent equations, full rank).
- Infinitely many solutions (dependent equations, free variables).
- No solution (inconsistent, e.g., parallel lines that never meet).

Graphically in 2D: intersection of lines.

**ELI5**

- Each equation is a straight-line rule ("twice the apples plus three oranges cost 10"). A solution is a shopping list (numbers of apples and oranges) that obeys every rule at once.

**Worked Example:**

**Solve the 2×2 system by substitution or elimination**

x + 2y = 8

3x - y = 3

1. Solve first for x: x = 8 - 2y.
2. Substitute into second: 3(8-2y) - y =3 => 24 -6y -y =3 => 24 -7y=3 => -7y=-21 => y=3.
3. x=8-2*3=2.
4. Unique solution (2,3). Verify in both originals.

### Common Mistakes

- **Inconsistent system detection**: A row [0 0 … 0 | c] with c≠0 means no solution; do not continue as if it were zero.
- **Free variables and parametrization**: When rank < number of variables, students forget to assign parameters to free variables and express basic variables in terms of them.
- **Augmented matrix column**: Forgetting to include the constants when setting up the augmented matrix for elimination.
- **Assuming unique solution always**: Many systems have none or infinite; check rank or row reduce fully.
- **Substitution vs elimination**: Both work; elimination often better for larger systems or when coefficients match.
- **Not verifying the solution**: Plug back into all original equations.

## Matrix (mathematics)

<!-- section: matrices-intro -->

Matrices are rectangular arrays that compactly represent linear systems, transformations, and data tables.

### Introduction to Matrices

A **matrix** is a rectangular array of numbers. An m × n matrix has m rows and n columns.

Matrices represent:
- Coefficients of linear systems (augmented matrix adds the constants column).
- Linear transformations (columns are images of basis vectors).
- Data tables (rows = samples, columns = features).

Special matrices: square, identity I (1s on diagonal), zero, diagonal, symmetric, triangular.

**ELI5**

- A matrix is just a tidy table of numbers. When the table comes from a system of equations, the rows are the equations and the columns are the coefficients of each variable.

**Worked Example:**

**Write the augmented matrix for the system**

2x - y + z = 4

x + 3y - 2z = -1

0x + y + 5z = 7

Augmented matrix:

[ 2  -1   1  |  4 ]

[ 1   3  -2  | -1 ]

[ 0   1   5  |  7 ]

### Common Mistakes

- **Matrix multiplication order and dimensions**: Always check inner dimensions match before multiplying; result size is outer dimensions.
- **Augmented matrix column**: Forgetting to include the constants when setting up the augmented matrix for elimination.
- **Confusing row and column vectors**: A 1×n is row vector; n×1 column.
- **Assuming all matrices square**: Many useful ones are not (e.g. data tables, transformations between different dim spaces).
- **Identity only for square**: I is square by definition.
- **Zero matrix multiplication**: 0*A=0 but A*0 may not be defined if dims wrong.

## Determinant

<!-- section: determinants -->

The determinant is a single number computed from a square matrix that tells you volume scaling, whether it flips space, and whether it is invertible.

### Determinants

The **determinant** of a square matrix is a single number that encodes volume scaling, orientation, and invertibility.

For 2×2: det[[a b],[c d]] = ad - bc.

For 3×3: cofactor expansion along a row: a11 C11 + a12 C12 + a13 C13 where Cij = (-1)^{i+j} times minor determinant.

Properties: det(AB)=det A det B, det(A^T)=det A, det(cA)=c^n det A, det I=1.

A matrix is invertible ⇔ det ≠ 0.

**ELI5**

- The determinant tells you "how much the matrix stretches or squishes space" and whether it flips orientation. Zero determinant means the transformation collapses everything onto a lower-dimensional flat (no volume).

**Worked Example:**

**Compute det of [[1,2,0],[3,1,1],[0,2,4]] by expansion along first row**

1. 1 * det[[1,1],[2,4]] - 2 * det[[3,1],[0,4]] + 0 * ...
2. Minor11: 1*4 -1*2=4-2=2. Cofactor + so +1*2=2.
3. Minor12: 3*4 -1*0=12. Cofactor - so -2*12 = -24.
4. Total: 2 - 24 = -22.

### Common Mistakes

- **Determinant sign errors in expansion**: The checkerboard pattern of +/− for cofactors is easy to mess up on 3×3 or larger.
- **For 2×2 swapping ad-bc vs ac-bd**: Always main diagonal minus anti.
- **Thinking det=0 means no matrix**: The matrix exists but is singular (not invertible).
- **Forgetting det(cA) = c^n det A**: The n is the size.
- **Computing det for non-square**: Only defined for square matrices.
- **Row reduction changes det**: Swaps flip sign, scaling a row by k multiplies det by k; adding multiple does not.

## Inverse matrix

<!-- section: inverses -->

The inverse matrix undoes the linear transformation represented by the original. It exists precisely when the determinant is nonzero.

### Matrix Inverses

The **inverse** A^{-1} of square A satisfies A A^{-1} = A^{-1} A = I.

For 2×2 [[a b],[c d]] with det=ad-bc ≠0:

A^{-1} = (1/det) [[d, -b], [-c, a]].

For larger matrices: use Gauss-Jordan (augment with I and row-reduce to [I | A^{-1}]) or adjugate formula (cofactors transposed / det).

If A x = b then x = A^{-1} b (when inverse exists).

**ELI5**

- The inverse is the "undo button" matrix. Multiplying by it reverses whatever the original matrix did, exactly like dividing by a number undoes multiplication.

**Worked Example:**

**Find the inverse of [[2,1],[5,3]] and verify**

1. det=2*3-1*5=6-5=1.
2. Inverse = 1/1 * [[3,-1],[-5,2]] = [[3,-1],[-5,2]].
3. Multiply: [[2,1],[5,3]] * [[3,-1],[-5,2]] = [[6-5, -2+2],[15-15, -5+6]] = [[1,0],[0,1]]. Good.

### Common Mistakes

- **Inverse formula for 2×2**: Forgetting the 1/det multiplier or swapping the diagonal entries incorrectly.
- **Row operations on wrong side**: When doing Gauss-Jordan for inverse, you must apply the same operations to the identity on the right.
- **Assuming every square matrix has inverse**: Only if det ≠0.
- **(A^{-1})^{-1} = A**: The inverse of the inverse is the original.
- **Inverse of product**: (AB)^{-1} = B^{-1} A^{-1} (reverse order).
- **Not checking det first**: Waste time if singular.

## Matrix multiplication

<!-- section: matrix-multiplication -->

Matrix multiplication corresponds to composition of linear transformations. It is defined via row-column dot products and is not commutative.

### Matrix Multiplication

The product AB is defined when the number of columns of A equals rows of B. The (i,j) entry of AB is the dot product of row i of A with column j of B.

Matrix multiplication is associative but not commutative: AB ≠ BA in general.

Multiplying by the identity leaves a matrix unchanged: AI = IA = A.

Matrix multiplication corresponds to composition of the linear transformations the matrices represent.

**ELI5**

- To multiply two tables, you "row-by-column dot products." Each entry in the result is a weighted sum that combines one row from the first table with one column from the second.

**Worked Example:**

**Compute the product [[1,2],[3,4]] * [[5],[6]]**

1. Result is 2×1.
2. First entry: 1*5 + 2*6 =5+12=17.
3. Second: 3*5 +4*6=15+24=39.
4. Product = [[17],[39]].

### Common Mistakes

- **Matrix multiplication order and dimensions**: Always check inner dimensions match before multiplying; result size is outer dimensions.
- **Assuming AB=BA**: Almost never true; order matters like function composition.
- **Forgetting the dot product definition**: Not just multiply corresponding entries; it's row of first with col of second.
- **Multiplying when inner dims don't match**: 2×3 * 2×4 undefined.
- **Confusing with elementwise (Hadamard) product**: AB is matrix product, not entry by entry.
- **Scalar vs matrix**: kA is scalar mult; different from multiplying by kI.

## Gaussian elimination

<!-- section: gaussian-elimination -->

Gaussian elimination (row reduction) is the systematic way to solve linear systems by operating on the augmented matrix until it is easy to read the answers.

### Gaussian Elimination

**Gaussian elimination** (or row reduction) uses three elementary row operations on the augmented matrix to reach row echelon form (or reduced row echelon form):

1. Swap two rows.
2. Multiply a row by a nonzero scalar.
3. Add a multiple of one row to another row.

These operations do not change the solution set.

Back-substitution on the resulting triangular system yields the solution(s). When a row becomes [0 0 … 0 | nonzero] the system is inconsistent.

**ELI5**

- Gaussian elimination is systematic "equation bookkeeping": you add and scale equations (rows) to cancel variables one column at a time until the system is easy to read from the bottom up.

**Worked Example:**

**Solve by Gaussian elimination**

x + y + z = 6

2x - y + z = 3

x + 2y - z = 0

Augmented:

[1 1 1 |6]

[2 -1 1 |3]

[1 2 -1 |0]

1. R2 = R2 - 2 R1: [0 -3 -1 | -9]

2. R3 = R3 - R1: [0 1 -2 | -6]

3. Now matrix:

[1 1 1 |6]

[0 -3 -1 |-9]

[0 1 -2 |-6]

4. Make pivot row2: multiply row2 by -1/3: [0 1 1/3 |3]

5. R3 = R3 -1* (new R2): [0 0 -7/3 | -9]

6. Back-sub: from last: z = (-9) / (-7/3) = 9*(3/7)=27/7.

Then y + (1/3)z =3 => y =3 - (1/3)(27/7)= (63-9)/7 =54/7.

Then x + y + z =6 => x=6 -54/7 -27/7=(42-81)/7= -39/7.

Solution: x=-39/7, y=54/7, z=27/7.

### Common Mistakes

- **Row operations on wrong side**: When doing Gauss-Jordan for inverse, you must apply the same operations to the identity on the right.
- **Free variables and parametrization**: When rank < number of variables, students forget to assign parameters to free variables and express basic variables in terms of them.
- **Inconsistent system detection**: A row [0 0 … 0 | c] with c≠0 means no solution; do not continue as if it were zero.
- **Forgetting that scaling a row scales the equation**: If you multiply row by 1/2 you are effectively dividing the equation.
- **Not using the pivot to clear above in RREF**: For full reduced, eliminate upward too.
- **Arithmetic slips in elimination**: Double check each row op.
