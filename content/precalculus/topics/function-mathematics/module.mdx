# Function (mathematics)

A function is a fundamental mathematical object that assigns to each input in its domain exactly one output. This precise relationship underpins almost all of precalculus, calculus, and higher mathematics.

## What Is a Function

<!-- section: what-is-a-function -->

A **function** $f$ from a set $A$ (the domain) to a set $B$ (the codomain) is a rule that assigns to every element $x \in A$ exactly one element $f(x) \in B$. We write this as $f: A \to B$. The "exactly one" condition is essential: no input may produce two different outputs.

Functions can be presented in many forms: an algebraic formula such as $f(x) = 2x + 1$, a table of input-output pairs, a graph in the coordinate plane, or even a verbal description. The graph of a function passes the vertical line test: no vertical line intersects the graph more than once.

Two functions are equal if they have the same domain and produce the same output for every input in that domain.

**ELI5**

- Think of a vending machine or a soda fountain button: press "A1" and you always get the same drink. If pressing A1 sometimes gave cola and sometimes orange, it would violate the function rule.
- In programming, a pure function is deterministic: same inputs always yield the same output, making it reliable and testable.

**Worked Example:**

**Determining whether a relation is a function from a set of points**

Given the set of ordered pairs: $\{(1,4), (2,4), (3,5), (1,6)\}$.

1. Group by input (first coordinate). Input 1 appears with outputs 4 and 6.
2. Because input 1 maps to two different outputs, this relation fails the definition of a function.
3. If we remove the pair (1,6), the remaining set $\{(1,4), (2,4), (3,5)\}$ is a function (different inputs may share the same output; that is allowed).

## Common Mistakes

- **Vertical line test misapplied on non-graphs**: The test only applies to graphs in the Cartesian plane; equations or sets of points require checking the definition directly.
- **Forgetting that different inputs may share outputs**: A function can send 1 and 2 both to 5; only the reverse (one input to two outputs) is forbidden.
- **Confusing a function with being one-to-one (injective)**: Not every function is invertible; $f(x)=x^2$ is a function but not one-to-one over all reals.
- **Ignoring the domain when claiming two formulas define the same function**: $f(x)=x/x$ and $g(x)=1$ agree where defined but have different domains.
- **Assuming every relation or equation defines y as a function of x**: Circles, vertical lines, and $x=y^2$ are classic counterexamples.
- **Forgetting that the codomain is part of the definition**: Two functions with same rule but declared different codomains are technically different (though range is what matters for outputs produced).
