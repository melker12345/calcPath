# Sets and Functions

Sets are the most fundamental objects in mathematics. Almost everything — numbers, functions, graphs, spaces — can be defined in terms of sets. Functions describe how elements of one set are mapped to elements of another; they are the primary tool for relating different collections of objects.

In computer science, sets model collections without duplicates (e.g., hash sets), while functions model mappings, transformations, and algorithms. Understanding when a function is invertible (bijective) is crucial for encryption, data compression, and database keys.

This chapter covers the language of sets, the algebra of set operations, the precise definition of functions, and the important special classes: injections, surjections, and bijections. We finish with power sets, which are central to counting and computability.

## Sets: Definitions and Notation

<!-- section: basic-sets -->

A **set** is an unordered collection of distinct objects (elements). We write $a \in S$ to mean "a is an element of S" and $a \notin S$ otherwise. Sets are usually denoted by capital letters and listed inside curly braces: $S = \{1, 3, 5\}$.

Order does not matter and duplicates are ignored: $\{1,2,2\} = \{2,1\}$.

The **empty set** $\emptyset = \{\}$ contains no elements. It is a subset of every set.

A set $A$ is a **subset** of $B$, written $A \subseteq B$, if every element of $A$ is also in $B$. We write $A \subset B$ for proper (strict) subset when $A \neq B$. The **universal set** $U$ is the set of all objects under consideration in a given context.

Two sets are **equal** if they contain exactly the same elements: $A = B \iff A \subseteq B$ and $B \subseteq A$.

**ELI5**

- A set is a club with no duplicate members and no ordering of members. "Alice and Bob are in the club" is the same as "Bob and Alice are in the club".
- The empty set is the club with zero members. It is still a perfectly valid club, and it is a "sub-club" of every other club.

### Worked Examples

**Worked Example:**

**Subset checks**

1. Is $\{1,3\} \subseteq \{1,2,3,4\}$? Yes, both elements appear.
2. Is $\{1,5\} \subseteq \{1,2,3,4\}$? No, 5 is missing.
3. $\emptyset \subseteq \{1,2\}$? Yes, vacuously — there are no elements in the empty set that fail to be in the bigger set.
4. $\{1,2\} \subseteq \{1,2\}$? Yes (every set is a subset of itself).

## Set Operations

<!-- section: operations -->

The basic operations on sets are:

- **Union**: $A \cup B = \{x \mid x \in A \text{ or } x \in B \text{ (or both)}\}$
- **Intersection**: $A \cap B = \{x \mid x \in A \text{ and } x \in B\}$
- **Set difference**: $A \setminus B = \{x \mid x \in A \text{ and } x \notin B\}$
- **Complement** (relative to universal set $U$): $A^c = U \setminus A = \{x \in U \mid x \notin A\}$

**Venn diagrams** are visual representations using overlapping circles. They make the distributive laws and De Morgan's laws for sets intuitive:

- De Morgan: $(A \cup B)^c = A^c \cap B^c$ and $(A \cap B)^c = A^c \cup B^c$

Key identities (all provable from definitions):

- $A \cup A = A$, $A \cap A = A$ (idempotent)
- $A \cup \emptyset = A$, $A \cap U = A$
- $A \cup B = B \cup A$ (commutative), likewise for $\cap$
- Distributivity: $A \cap (B \cup C) = (A \cap B) \cup (A \cap C)$
- $|A \cup B| = |A| + |B| - |A \cap B|$ (inclusion-exclusion for two sets)

**ELI5**

- Union is "everyone from either club" (merge the membership lists, remove duplicates).
- Intersection is "people who belong to both clubs".
- Difference is "members of A who are not in B".
- Complement is "everyone in the universe who is not in A".

### Worked Examples

**Computing with small sets**

Let $A = \{1,2,3,4\}$, $B = \{3,4,5,6\}$, $U = \{1,2,3,4,5,6,7\}$.

- $A \cup B = \{1,2,3,4,5,6\}$
- $A \cap B = \{3,4\}$
- $A \setminus B = \{1,2\}$
- $B \setminus A = \{5,6\}$
- $A^c = \{5,6,7\}$
- $|A \cup B| = 4 + 4 - 2 = 6$ (matches listing).

**Proving a set identity**

Prove $(A \cup B) \cap (A \cup C) = A \cup (B \cap C)$ (distributivity of union over intersection, one direction).

Left side: x in both (A∪B) and (A∪C) means (x in A or B) and (x in A or C).

Case analysis or logic: if x in A, then x is in right side. If x not in A, then must be in B and in C, so in B∩C, hence right side. (Symmetric argument the other way.)

## Functions: Definition and Properties

<!-- section: functions -->

A **function** $f: A \to B$ (from domain A to codomain B) is a rule that assigns to every element $x \in A$ exactly one element $f(x) \in B$.

- The **image** (range) of f is $\{f(x) \mid x \in A\} \subseteq B$.
- The **preimage** of a set $C \subseteq B$ is $f^{-1}(C) = \{x \in A \mid f(x) \in C\}$.

Functions must be **total** (defined for every element of the domain) and **single-valued**.

In set theory a function is formally a set of ordered pairs $(x,y)$ with no two pairs sharing the same first component but different seconds.

**ELI5**

- A function is a perfect "machine" or "vending rule": drop in any input from the domain and it always spits out exactly one output in the codomain. No input left out, no input giving two different answers.
- Think of $f(x) = x^2$ from reals to reals: every real goes in, a nonnegative real comes out.

### Worked Examples

**Checking whether a relation is a function**

1. Relation on $\{1,2,3\}$: pairs (1,4), (2,4), (3,5). This is a function from {1,2,3} to {4,5} (each input has exactly one output).
2. Pairs (1,4), (1,5), (2,3): not a function — input 1 maps to two values.
3. Pairs (1,4), (2,5): not a function from {1,2,3} because 3 has no output (not total).

## Injective and Surjective Functions

<!-- section: injective-surjective -->

- **Injective** (one-to-one): $f(x_1) = f(x_2)$ implies $x_1 = x_2$. Different inputs give different outputs. No two arrows point to the same target.
- **Surjective** (onto): Every element of the codomain is the image of at least one element in the domain. The function "covers" the entire codomain.
- For finite sets, an injection from A to B implies $|A| \leq |B|$; a surjection implies $|A| \geq |B|$.

To test injectivity: assume $f(x_1)=f(x_2)$ and prove $x_1=x_2$ (often by algebra or contradiction).

To test surjectivity: for arbitrary $y \in B$, solve $f(x)=y$ and show a solution $x$ exists in A.

**ELI5**

- Injective: like a unique ID — no two people share the same fingerprint produced by the function.
- Surjective: like every seat in the theater is occupied by at least one audience member mapped by "who sits where".

### Worked Examples

**Linear function example**

Consider $f: \mathbb{R} \to \mathbb{R}$, $f(x) = 3x + 1$.

- Injective: Suppose $3x_1 + 1 = 3x_2 + 1$. Subtract 1: $3x_1 = 3x_2$ ⇒ $x_1 = x_2$. Yes.
- Surjective: For any real y, solve 3x + 1 = y ⇒ x = (y-1)/3, which is always real. Yes.
- Therefore bijective; inverse is $f^{-1}(y) = (y-1)/3$.

**Non-surjective example**

$f: \mathbb{Z} \to \mathbb{Z}$, $f(n) = n^2$.

- Injective: No (e.g. f(2)=f(-2)).
- Surjective? No — negative numbers and most positives never hit.

## Bijective Functions and Inverses

<!-- section: bijections -->

A function is **bijective** if it is both injective and surjective. Equivalently, it has a two-sided inverse function $f^{-1}: B \to A$ such that $f(f^{-1}(y)) = y$ for all y and $f^{-1}(f(x)) = x$ for all x.

For finite sets of the same size, any injection is automatically a surjection (and vice versa) — the pigeonhole principle forces it.

Bijections establish that two sets have exactly the same "size" (cardinality), even if infinite.

**ELI5**

- Bijective is a perfect matching: every input has a unique output and every output is used exactly once. Like a perfect pairing of dance partners with no one left out and no one dancing with two people.
- The inverse simply swaps the direction of every arrow.

### Worked Examples

**Showing a function is bijective**

Let $f: \mathbb{R}^+ \to \mathbb{R}^+$, $f(x) = 1/x$ (positive reals).

- Injective: 1/x1 = 1/x2 ⇒ x1 = x2 (cross multiply, positives).
- Surjective: for any y > 0, x=1/y >0 is in domain and f(x)=y.
- Inverse is itself: f(f(x)) = x.

**Finite set counting**

A set with 5 elements can be mapped bijectively onto any other 5-element set. There are 5! = 120 such bijections (permutations).

## Power Sets and Cardinality

<!-- section: power-sets -->

The **power set** of S, written $\mathcal{P}(S)$ or 2^S, is the set of all subsets of S.

- If S is finite with n elements, $|\mathcal{P}(S)| = 2^n$.
- This includes $\emptyset$ and S itself.
- Power sets grow extremely fast: a 10-element set has 1024 subsets; 20 elements → over a million.

For infinite sets, Cantor's theorem says $|\mathcal{P}(S)| > |S|$ always (no set is in bijection with its power set). This is the basis of the idea that there are "different sizes" of infinity.

**ELI5**

- The power set is "the club of all possible sub-clubs you can form from the original club members", including the empty club and the full club.
- Each element can be "in or out" of a subset independently — hence 2 choices per element, 2^n total subsets.

### Worked Examples

**List the power set**

Let S = {a,b}.

- $\mathcal{P}(S) = \{ \emptyset, \{a\}, \{b\}, \{a,b\} \}$. Exactly 4 = 2^2 elements.

**Cardinality formula application**

A set with 0 elements: power set has 1 element (just empty).

n=1: 2 subsets.

n=2: 4.

n=3: 8.

## Common Mistakes

- Confusing $\subseteq$ (subset, allows equal) with $\subset$ (sometimes used for proper).
- Thinking $\emptyset$ is the same as $\{\emptyset\}$ — the latter has one element.
- Forgetting that union and intersection are commutative but difference is not: $A \setminus B \neq B \setminus A$ in general.
- Believing every function has an inverse — only bijections do.
- Confusing codomain with range (image). The codomain is declared; the image may be smaller.
- Assuming a function from a larger finite set to a smaller one can be injective (pigeonhole forbids it).
- Writing $f(x) \in A \to B$ instead of $f: A \to B$.
- Forgetting the empty set when counting subsets.
