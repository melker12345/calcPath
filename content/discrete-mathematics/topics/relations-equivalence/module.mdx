# Relations and Equivalence

While sets collect objects, relations describe how objects stand to one another. The most important relations are those that behave like "equality" (equivalence relations) and those that behave like "less than or equal" (partial and total orders). These concepts underlie modular arithmetic, database normalization, sorting, topological sorting in compilers, and the ordering of tasks in project management.

A relation is simply a set of ordered pairs, yet the properties we impose on it — reflexivity, symmetry, transitivity, antisymmetry — turn it into a powerful organizational tool.

## Binary Relations

<!-- section: relations -->

A **binary relation** R on a set A is defined as a subset of the Cartesian product $A \times A$. We write $a R b$ (or $(a,b) \in R$) to mean that a is related to b.

- The **domain** of R is the set of all first components that appear.
- The **range** (or codomain usage) is the set of second components.

Relations can be represented as:
- Lists of pairs
- Directed graphs (nodes = elements of A, arrow a→b when aRb)
- 0-1 matrices (rows and columns indexed by A, 1 at (i,j) if i R j)

Examples: "is parent of", "is greater than", "divides", "is congruent modulo m".

**ELI5**

- A relation is a directed "friendship" or "arrow" between members of the same club. Some people point to themselves (loops), some point one way, some both ways.
- Not every pair needs an arrow; the relation picks which connections exist.

### Worked Examples

**Worked Example:**

**Matrix representation**

Let A = {1,2,3}. Let R = {(1,1),(1,2),(2,1),(3,3)}.

The matrix is:

1 1 0

1 1 0

0 0 1

This relation is reflexive on 1 and 3 but not on 2.

## Properties of Relations

<!-- section: properties -->

Important properties (for a relation R on A):

- **Reflexive**: for every a in A we have (a,a) in R. (Everyone relates to self; diagonal of matrix is all 1s.)
- **Irreflexive**: ∀a, (a,a) ∉ R. (No self-loops.)
- **Symmetric**: aRb ⇒ bRa. (Arrows come in both directions or none.)
- **Antisymmetric**: aRb and bRa ⇒ a = b. (The only two-way arrows are self-loops.)
- **Transitive**: aRb and bRc ⇒ aRc. (Arrows compose: if path of length 2 then direct arrow.)
- **Total (connected)**: for all a ≠ b, either aRb or bRa (or both).

These are independent in general; combinations produce the named classes of relations we care about.

**ELI5**

- Reflexive: "I am the same age as myself" — always true for equality-like things.
- Symmetric: "is a sibling of" — if Alice is sibling of Bob, Bob is sibling of Alice.
- Transitive: "is an ancestor of" — if A ancestor of B and B of C, then A of C.
- Antisymmetric: "is less than or equal" — if a ≤ b and b ≤ a then a must equal b.

### Worked Examples

**Checking properties on a small relation**

R on {a,b,c}: pairs (a,a), (b,b), (c,c), (a,b), (b,a), (a,c).

- Reflexive? Yes (all diagonals).
- Symmetric? No — (a,c) present but not (c,a).
- Transitive? a R b and b R a (ok); a R c — no further from c. But since no (b,c) etc., actually holds vacuously for most. Wait: aRb and bRa ok symmetric. aRc and ? No violation found, yes transitive here.
- Antisymmetric? aRb and bRa but a≠b — violates antisymmetric.

## Equivalence Relations

<!-- section: equivalence -->

An **equivalence relation** is one that satisfies reflexivity, symmetry, and transitivity all at once.

It captures the idea of "being the same in some respect" or "indistinguishable for our purposes".

Famous examples:
- Equality itself.
- Congruence modulo m: a ≡ b (mod m) iff m divides (a-b).
- "Has the same birthday" on people (assuming no leap issues).
- "Is the same connected component" in a graph.

**ELI5**

- Equivalence is "sameness" that is consistent: if I'm the same type as you, you're the same as me (sym), I'm same as myself (refl), and if you're same as her then I am too (trans).
- It lets us treat whole groups as single "super elements".

### Worked Examples

**Congruence mod 3**

On integers: a ~ b iff a - b divisible by 3.

- 7 ~ 1 because 6 divisible by 3.
- Reflexive: a-a=0 divisible by 3.
- Symmetric: if a-b div by 3 then b-a = -(a-b) also.
- Transitive: obvious from adding differences.
- Classes: ..., [-2],[-1],[0],[1],[2] where [k] = {n | n ≡ k mod 3}.

## Equivalence Classes and Partitions

<!-- section: partitions -->

Given equivalence relation ~ on A, the **equivalence class** of a is [a] = {b ∈ A | b ~ a}.

Key facts:
- Every element belongs to exactly one class (its own).
- Classes are either equal or disjoint.
- The set of all distinct equivalence classes forms a **partition** of A: the blocks are nonempty, pairwise disjoint, and cover A.

Conversely, every partition of A defines an equivalence relation: a ~ b iff a and b are in the same block of the partition.

The set of equivalence classes is called the **quotient set** A / ~ .

**ELI5**

- Each equivalence class is a "bucket" or "team". Everyone in a bucket is equivalent; different buckets are distinct.
- The buckets cover the whole group and don't overlap — a clean division of the club into teams.

### Worked Examples

**Partition from mod 4**

Integers modulo 4: four classes [0],[1],[2],[3].

E.g. [2] = {..., -2,2,6,10,...}.

These four classes partition ℤ; no integer is in two, and every integer is in one.

## Partial Orders and Posets

<!-- section: partial-orders -->

A **partial order** (or poset) is a relation satisfying reflexivity, antisymmetry, and transitivity.

We often write ≤ or | (divides) or ⊆ .

Not every pair of elements needs to be comparable. When a and b have neither a ≤ b nor b ≤ a they are **incomparable**.

Examples:
- Subset inclusion on the power set of any set.
- Divisibility on positive integers (1 divides everything; primes are minimal above 1).
- "Is a prerequisite for" on university courses (some courses have no prereq relation either way).
- Task dependency graphs in project management (DAGs induce posets).

A **poset** is a set equipped with a partial order.

**ELI5**

- Partial order is "can be lined up some but not all ways". Some people are taller than others, but two people of same height are equal; two unrelated people may not have height order in our data.
- "Divides" : 2 divides 8, 4 divides 8, but 3 and 4 are incomparable under divides.

### Worked Examples

**Hasse diagram (visual of poset)**

For subsets of {1,2} under ⊆ :

Bottom: ∅

Middle level: {1} and {2} (both cover ∅, incomparable to each other)

Top: {1,2} covers both middles.

No direct line between {1} and {2} because incomparable.

This is the Boolean lattice of rank 2.

## Total Orders

<!-- section: total-orders -->

A **total order** (linear order, chain) is a partial order where every pair of elements is comparable (for any a, b either aRb or bRa).

Examples: the usual ordering of numbers, alphabetical order on strings, chronological order on dates.

In a total order there are no "branches"; everything can be lined up in a single sequence (possibly with ties if not strict).

A **well-order** is a total order in which every nonempty subset has a least element (e.g., natural numbers under ≤; not the integers or reals).

**ELI5**

- Total order is a complete ranking: first place, second, third... with no two people left "side by side with no ranking between them".
- Partial allows side-by-side incomparable items; total forces a decision on every pair.

### Worked Examples

**Extending a partial order to a total order**

The poset of {a,b,c} with a ≤ b, a ≤ c (b and c incomparable) can be extended to two different total orders:

1. a ≤ b ≤ c
2. a ≤ c ≤ b

Both respect the original relations. (Topological sorts of the Hasse diagram.)

## Common Mistakes

- Confusing symmetric with antisymmetric. "Is sibling of" is symmetric; "≤" is antisymmetric. They are almost opposites.
- Forgetting reflexivity when checking equivalence or partial orders (many students remember sym+trans but drop refl).
- Thinking every equivalence relation is equality — no, it can collapse many elements into one class (modular arithmetic).
- Believing that in a poset every pair must be comparable — that would make it total. Partial allows incomparables.
- Mixing up "partition" (the blocks) with the equivalence relation itself.
- Forgetting that antisymmetry only kicks in when both directions hold: it forces a=b, but allows one-way relations freely.
- Assuming the inverse of a relation automatically inherits the same properties (symmetry yes, transitivity no in general).
