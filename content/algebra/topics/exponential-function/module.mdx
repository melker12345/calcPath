# Exponential function

An exponential function has the form f(x) = a * b^x where a ≠ 0, b > 0, b ≠ 1, and the variable x appears in the exponent.

Exponential functions model phenomena with constant percent growth or decay: populations, compound interest, radioactive decay, cooling, and many natural processes. Their graphs are smooth curves that are always positive, have a horizontal asymptote, and grow (or decay) faster and faster.

## Definition and Basic Shape

<!-- section: definition-and-basic-shape -->

f(x) = a b^x .

- a is the initial value or y-intercept (f(0) = a).
- b is the growth/decay factor (the base).
- The graph is continuous, always above the x-axis, and has horizontal asymptote y=0.

For b > 0, b ≠ 1 the function is one-to-one, so it has an inverse (the logarithm).

**ELI5**

- Each time x increases by 1, the output is multiplied by b (not added).
- "Percent change is constant" is the key: +50% each step means *1.5, no matter the current size.

**Worked Example:**

1. $f(x) = 200 \cdot (1.1)^x$: starts at $200$ when $x=0$, grows $10\%$ per unit $x$.
2. At $x=1$: $220$; $x=2$: $242$, etc.

**Worked Example:**

1. $g(x) = 50 \cdot (0.8)^x$: starts at $50$, decays $20\%$ per unit time, approaches $0$.

## Growth versus Decay and the Role of the Base

<!-- section: growth-vs-decay-and-base -->

If b > 1: exponential growth. As x → +∞, f(x) → +∞ (if a>0); as x → -∞, f(x) → 0.

If 0 < b < 1: exponential decay. As x → +∞, f(x) → 0; as x → -∞, f(x) → +∞ (a>0).

The base b is often written 1 + r where r is the growth rate (r > -1, r ≠ 0).

**ELI5**

- b >1 is "more than one copy each time step" → explodes upward.
- b <1 is "less than one copy" → shrinks toward zero.

**Worked Example:**

1. Compound interest: $A = P (1 + r/n)^{n t}$ is exponential in $t$.

**Worked Example:**

1. Half-life: amount = initial $\cdot (1/2)^{t / \text{half-life}}$, $b=1/2 <1$, decay.

## Transformations of Exponential Functions

<!-- section: transformations -->

The usual transformations apply:

- f(x) + k : vertical shift.
- a f(x) : vertical stretch/compression/reflection (sign of a).
- f(x - h) : horizontal shift.
- f(c x) : horizontal stretch/compression (c affects the "speed" of growth).

The horizontal asymptote moves with vertical shifts. The y-intercept changes with a and shifts.

**ELI5**

- Shifting and stretching work the same as for other functions, but the exponential shape stays recognizable.
- Horizontal compression makes it grow faster.

**Worked Example:**

1. $g(x) = -2 \cdot 3^{x-1} + 5$: reflect over $x$-axis, stretch vert by 2, shift right 1, up 5. Asymptote $y=5$, $y$-intercept $g(0) = -2 \cdot 3^{-1} +5 = -2/3 +5 = 13/3$.

## Solving Basic Exponential Equations

<!-- section: solving-basic-equations -->

When both sides can be written as powers of the same base, equate the exponents.

Otherwise take logarithms (see logarithm lesson) or use numerical/graphical methods.

Always consider domain (expression defined) and check solutions.

**ELI5**

- If the bases match, the exponents must match (because exponential is one-to-one).
- "Undo" the exponential with a log when bases don't match.

**Worked Example:**

1. Solve $4^x = 16$.
2. $4^x = 4^2$ (since $16=4^2$), so $x=2$.

**Worked Example:**

1. Solve $5^{x-1} = 125$.
2. $125=5^3$, so $x-1=3$, $x=4$.

## Common Mistakes

- **Confusing exponential with polynomial (x in base vs exponent)**: 2^x grows much faster than any x^n.
- **Thinking b=1 or b<0 is allowed for real exponential functions in basic algebra**: Base must be positive and not 1 for the standard real exponential.
- **Forgetting the horizontal asymptote when sketching or transforming**: It moves only with vertical shifts.
- **Solving  b^x = c by taking xth root incorrectly**: Better to express c as power of b when possible.
- **Sign errors with negative a**: a can be negative, which reflects the graph over x-axis (still positive? No, then it goes negative, but b^x always >0 so sign follows a).
- **Ignoring that for 0<b<1 the growth/decay labels flip with direction of x**.
- **Writing (a b)^x = a^x * b^x correctly but then mishandling when a is the variable**.
