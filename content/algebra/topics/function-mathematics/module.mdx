# Function (mathematics)

A function is a special kind of relation: it pairs every input with exactly one output. This "exactly one" rule is what makes functions predictable and powerful for modeling the real world.

Functions appear as formulas, graphs, tables, and verbal descriptions. Once you understand the definition you can decide whether any given relation qualifies as a function and can use function notation fluently.

## Precise Definition

<!-- section: definition -->

A function f from a set A (the domain) to a set B is a rule that assigns to each element x in A exactly one element f(x) in B.

No input may have two different outputs. Two different inputs may share the same output.

**ELI5**

- A function is a machine with a strict rule: drop in one number, get exactly one number out. No arguing, no choices.
- Vending machines are functions (one button → one item); a "choose any snack for $1" machine would not be.

**Worked Example:**

The relation "y is the square of x" is a function: each x gives one y = x². The relation "y is a square root of x" is not a function for x > 0 because two outputs exist.

## The Vertical Line Test

<!-- section: vertical-line-test -->

For graphs in the coordinate plane: if any vertical line crosses the graph more than once, the graph does not represent a function.

A vertical line at a fixed x would intersect two points with the same x but different y's, violating the one-output rule.

**ELI5**

- Imagine a vertical stick sliding across the graph. If it ever touches the curve in two places at once, that x has two y's and is not a function.
- Circles, most sideways parabolas, and figure-eights all fail the test.

**Worked Example:**

The graph of y = x² passes (any vertical line hits at most once). The graph of x = y² fails (vertical lines through the right half hit twice).

## Function Notation

<!-- section: function-notation -->

f(x) names the output of the function named f when the input is x.

It is read "f of x." It does not mean multiplication.

Evaluating: replace every occurrence of the input variable with the given number and simplify.

**ELI5**

- f is the name of the recipe. The parentheses hold the ingredient you are feeding it.
- f(3) asks "what does the recipe give when I use 3?"

**Worked Example:**

Let f(x) = 2x² − 5x + 1.

f(2) = 2(4) − 5(2) + 1 = 8 − 10 + 1 = −1.

f(−1) = 2(1) − 5(−1) + 1 = 2 + 5 + 1 = 8.

## Everyday and Mathematical Examples

<!-- section: examples -->

- Area of a square: A(s) = s² (one side → one area).
- Temperature conversion: C(F) = (F − 32) × 5/9.
- Cost of buying n apples at $0.50 each: C(n) = 0.50n (plus any fixed fee).
- The rule "the output is the input plus 3" is the function f(x) = x + 3.

**ELI5**

- Many real quantities depend on exactly one other quantity in a rule-like way; that dependence is a function.
- Once named, the function can be used, composed, graphed, and studied without rewriting the rule every time.

**Worked Example:**

The cost of a taxi ride that charges $2.50 per mile is the function C(m) = 2.5m. C(4) = 10 dollars for a 4-mile ride. Different distances give different costs; each distance gives exactly one cost.

## Common Mistakes

- **Thinking f(x) means f multiplied by x**: It is function application, not multiplication.
- **Believing that every relation is a function**: Many are not (circles, one-to-many tables).
- **Forgetting that two inputs can map to the same output**: Functions are allowed to be many-to-one; they are forbidden only from being one-to-many.
- **Confusing domain with range**: Domain = inputs you may feed in; range = outputs that actually come out.
- **Assuming the vertical line test applies only to continuous curves**: It applies to any graph, including discrete points.
