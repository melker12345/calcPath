# Jacobi Symbol

The Jacobi symbol (a/n) generalizes the Legendre symbol to odd composite n: if n= product p_i^{e_i} odd, (a/n) = product (a/p_i)^{e_i} (Legendre).

It is completely multiplicative in a, and (a/n)=0 if gcd(a,n)>1.

Unlike Legendre, (a/n)=1 does not imply a is quadratic residue mod n (it can be 1 even if non-residue mod some p| n, as long as the product of signs is +1).

It is still useful because it can be computed using a reciprocity law similar to Legendre, in time comparable to gcd, without factoring n.

Used in the Solovay-Strassen primality test (a probabilistic test using the Jacobi symbol).

## Definition

<!-- section: jacobi-def -->

For odd positive integer n>1 and integer a, the **Jacobi symbol** (a/n) is defined by factoring n into odd primes (with multiplicity) and multiplying the Legendre symbols:

If n = ∏ p_i ^{e_i}, then (a/n) = ∏ (a / p_i )^{e_i} where ( / p_i) are Legendre.

If gcd(a,n)>1 then (a/n)=0.

For n=1, (a/1)=1 by convention.

**ELI5**

- It is the "product of the yes/no answers" for the prime power factors of n.
- It can say +1 even when there is no square root mod n, because the "no" from one prime can be canceled by "yes" from another.

**Worked Example:**

**(2/15)**

15=3*5.

(2/3)=-1 (3=3 mod8), (2/5)= (2/5), 5=5 mod8=5≡-3, -1? 5 mod8=5, the rule (2/p)=1 if p≡1 or7 mod8, -1 if 3 or5 mod8.

So (2/5)=-1.

(2/15)= (-1)*(-1)=1.

But is 2 square mod15? x^2≡2 mod3 and mod5. Mod3 2≡-1, squares mod3 0,1 not 2. No solution mod3, hence no mod15. So (2/15)=1 but not residue. Classic example why Jacobi=1 does not imply residue.

## Properties and Reciprocity

<!-- section: jacobi-properties -->

The Jacobi symbol satisfies:

(a/n) = (b/n) if a≡b mod n.

Completely multiplicative in a: (ab/n)=(a/n)(b/n).

(a/n)=(a mod n /n).

For the "quadratic reciprocity" for Jacobi: if m,n odd positive coprime, then (m/n)(n/m) = (-1)^{(m-1)/2 (n-1)/2}.

Same as for primes.

Also the rules for ( -1 /n ) = (-1)^{(n-1)/2}, (2/n)= (-1)^{(n^2-1)/8}.

These allow computing (a/n) by a Euclidean-like algorithm (repeatedly flip using reciprocity, reduce mod, factor out 2 and -1 using the rules), without ever factoring n.

The running time is O(log^2 n) like gcd.

**ELI5**

- The reciprocity law works for the composite Jacobi just as for Legendre, letting you swap the arguments with a sign.
- Combined with the special rules for 2 and -1, you can reduce the size like Euclid algorithm for gcd, computing the symbol fast.

## Use in Primality Tests

<!-- section: jacobi-primality -->

The Solovay-Strassen test: for random a, compute the Jacobi (a/n) and a^{(n-1)/2} mod n. If n prime they are equal by Euler criterion (the power equals Legendre=Jacobi).

If for some a they differ, n composite.

If they agree for many a, declare probable prime.

It detects Carmichael numbers (for which the power is 1, but Jacobi may differ for some a).

**ELI5**

- For prime, the power must equal the Jacobi symbol.
- If for a witness a the power mod n does not equal the easily computed Jacobi, then n cannot be prime.

**Worked Example:**

**For a Carmichael like 561, there exist a where a^{560} ≡1 mod561 but (a/561) ≠1, detecting compositeness.**

Yes.

## Common Mistakes

- **(a/n)=1 implies quadratic residue mod n**: False for composite n; it is necessary but not sufficient.
- **Jacobi requires factoring n**: No, the reciprocity algorithm computes it without factoring n (that's its point).
- **Jacobi defined for even n**: No, n must be odd positive.
- **Confusing with Legendre**: Legendre is the special case when n prime; Jacobi can be 1 for non-residues.
