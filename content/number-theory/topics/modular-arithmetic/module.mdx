# Modular Arithmetic

Modular arithmetic is arithmetic on the "clock": after reaching the modulus you wrap around. Formally, we work in the ring of residue classes $\mathbb{Z}/n\mathbb{Z}$. It lets us ignore multiples of the modulus and focus only on remainders, which simplifies calculations involving divisibility, cyclic patterns, and large exponents.

This is the language of modern cryptography (Diffie-Hellman, RSA, elliptic curves), checksums, hashing, and computer science (hash tables, cyclic buffers). Many Diophantine problems and congruences become tractable only after translating them into modular equations.

Key concepts: congruence relation, arithmetic operations that are well-defined on classes, inverses when they exist, and the Chinese Remainder Theorem for combining information from several moduli.

## Congruences and Residue Classes

<!-- section: congruences-residues -->

We say $a \equiv b \pmod{m}$ (a is congruent to b modulo m) if and only if $m$ divides $(a-b)$, i.e., $a - b = m k$ for some integer $k$.

This is an equivalence relation on $\mathbb{Z}$: reflexive, symmetric, transitive.

The equivalence classes are the residue classes $[0], [1], \dots, [m-1]$. We usually represent each class by its least nonnegative residue $r$ with $0 \leq r < m$.

Addition, subtraction, and multiplication are well-defined on classes: if $a\equiv a' \pmod m$ and $b\equiv b' \pmod m$ then $a+b \equiv a'+b'$, $a-b \equiv a'-b'$, and $a b \equiv a' b' \pmod m$.

**ELI5**

- On a 12-hour clock, 10 o'clock + 5 hours is 3 o'clock, not 15. 15 ≡ 3 mod 12 because they differ by a multiple of 12 (the full cycles).
- All numbers that leave the same remainder when divided by m are "the same" for purposes of mod-m arithmetic.

**Worked Example:**

**Compute 17 + 25 mod 12**

17 ≡ 5 mod 12 (17-12=5)

25 ≡ 1 mod 12 (24+1)

5+1=6, so 17+25 ≡ 6 mod 12.

(42 mod 12 = 6, yes.)

## Modular Inverses and Linear Congruences

<!-- section: inverses-congruences -->

An integer a has a **multiplicative inverse** modulo m (written $a^{-1} \bmod m$) if there exists x such that a x ≡ 1 mod m. This exists if and only if gcd(a,m)=1 (a and m coprime).

To solve the linear congruence a x ≡ b mod m:

- Compute d = gcd(a,m).
- Solutions exist iff d divides b.
- If so, there are exactly d distinct solutions modulo m.
- First find one particular solution using extended gcd (scale), then add multiples of m/d.

**ELI5**

- The inverse is the "undo" multiplier: multiply by it and you get back to 1 (mod m). Only possible when a and m share no common factors other than 1.
- Solving ax ≡ b mod m is like "divide both sides by a" but you must check the gcd condition and accept multiple answers when d>1.

**Worked Example:**

**Solve 5x ≡ 3 mod 7**

gcd(5,7)=1 divides 3.

Extended gcd: 5*3 +7*(-2)=1 ⇒ 5*3 ≡1 mod7, so inverse of 5 is 3.

Multiply both sides by 3: x ≡ 9 ≡ 2 mod7.

Check: 5*2=10≡3 mod7 yes.

Only one solution mod 7.

## Powers and Euler/Fermat (preview)

<!-- section: modular-powers -->

When computing large powers a^k mod m we never compute the full number; instead reduce modulo m at each step (or use repeated squaring).

By Fermat's Little Theorem (when p prime, gcd(a,p)=1): a^{p-1} ≡ 1 mod p, so a^k mod p can be reduced by exponent mod (p-1).

More generally Euler's theorem uses φ(m).

**ELI5**

- Exponents wrap around in the exponent "clock" whose size is related to how many invertible residues there are.
- You can reduce the exponent first using the theorem, making huge exponents trivial.

**Worked Example:**

**Compute 3^{100} mod 7 using Fermat (preview)**

By Fermat, since 7 prime, 3^6 ≡1 mod7.

100 = 16*6 + 4, so 3^{100} = (3^6)^{16} * 3^4 ≡ 1^{16} * 81 ≡ 4 mod7 (81/7=11*7=77, 81-77=4).

## The Ring Z/nZ

<!-- section: ring-structure -->

The set of residue classes with + and * forms a ring $\mathbb{Z}/n\mathbb{Z}$. It is a field precisely when n is prime (every nonzero element has inverse).

When n composite the ring has zero-divisors: e.g. mod 6, 2*3≡0 but neither is 0 mod6.

This algebraic structure explains why some equations have more or fewer solutions mod n than over the reals.

**ELI5**

- Modulo a prime everything (nonzero) has an inverse, like the usual numbers. Modulo composite some numbers "eat" others to give zero, which has no inverse.
- The ring view lets us import all the usual algebra rules (distributivity etc.) while remembering the wrap-around.

**Worked Example:**

**Zero divisors mod 6**

2*3=6≡0 mod6.

Neither 2 nor 3 is ≡0. In the ring they multiply to the zero element.

## Common Mistakes

- **Reducing only one side**: You must reduce the entire expression; 5 + 7 ≡ 0 mod 6 is true, but writing 5+7=12 then 12 mod6 separately is ok only if you do it correctly.
- **Inverses exist for all a**: Only when coprime to m. 2 has no inverse mod 4 (2x even, never 1 mod4).
- **Number of solutions to ax≡b mod m**: Exactly d=gcd(a,m) solutions when d|b, not always 1 or always 0.
- **Negative remainders**: We conventionally use 0 ≤ r < m. -1 ≡ m-1 mod m is the standard positive representative.
- **Confusing ≡ with =**: a ≡ b mod m does not mean a=b; it means they differ by multiple of m. Never drop the "mod m" when it matters.
