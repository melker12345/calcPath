# Linear Diophantine Equations

A linear Diophantine equation is ax + by = c with a,b,c integers, seeking integer solutions x,y.

It has solutions iff gcd(a,b) divides c.

The general solution can be found from one particular solution using the extended Euclidean algorithm, then adding the homogeneous solutions (multiples of b/d and -a/d).

This is the most basic Diophantine equation and the starting point for the theory of linear equations in more variables and for the geometry of numbers.

## Solvability Condition

<!-- section: linear-dioph-solvability -->

The equation a x + b y = c has integer solutions x,y if and only if d = gcd(a,b) divides c.

Proof: if solution, then d divides left hence right.

Conversely, if d|c, use extended Euclid to find x0,y0 with a x0 + b y0 = d, then multiply by c/d: a (x0 c/d) + b (y0 c/d) = c.

**ELI5**

- The left side a x + b y is always a multiple of the greatest common "step" d=gcd(a,b). So c must be reachable, i.e. multiple of d.
- If yes, then you can reach it.

**Worked Example:**

**Solve 6x + 9y = 3**

d=3 divides 3.

Extended: 6*(-1) + 9*1 =3?  -6+9=3 yes.

So x0=-1, y0=1 for =3.

Particular for =3: same (c/d=1).

General: x = -1 + (9/3) t = -1 +3 t

y = 1 - (6/3) t =1 -2 t.

Check: 6(-1+3t) +9(1-2t)= -6+18t +9 -18t =3 yes.

## Finding All Solutions

<!-- section: linear-dioph-general -->

Once you have one particular solution (x0,y0) to a x + b y = c, the general is

x = x0 + (b/d) t

y = y0 - (a/d) t

for t integer.

This parametrizes all solutions; the coefficients come from the kernel of the map (x,y) |→ a x + b y, which is generated by (b/d, -a/d).

**ELI5**

- From one solution, you can add "full cycles" of the smaller coefficient and subtract from the other to stay at the same total c.
- The step size is b/d and a/d because dividing by d gives the primitive equation.

**Worked Example:**

**From the particular (-1,1) for 6x+9y=3, the general is x=-1+3t , y=1-2t as above.**

For t=0: -1,1

t=1: 2, -1 : 6*2 +9*(-1)=12-9=3 yes.

## Applications to Frobenius and Chicken Nugget

<!-- section: linear-dioph-applications -->

For two coprime a,b the largest number not representable as nonnegative ax+by is a b -a -b (Frobenius coin problem).

For more than two variables it is hard (NP-hard in general).

In programming and contest math, the linear Diophantine is used to solve "buy exactly c items with packs of a and b".

**ELI5**

- With coins of value a and b coprime, the amounts you cannot make are finite; the biggest is ab-a-b.
- Once you can make d consecutive amounts (d=max(a,b)), you can make all larger by adding the small coin.

**Worked Example:**

**Frobenius for 4 and 7: 4*7-4-7=17. Indeed 17 cannot, 18= 1*4 +2*7? Wait 4+14=18, 19=3*4 +1*7=12+7=19, 20=5*4, 21=3*7, etc. 17 no combination of nonneg.**

## Common Mistakes

- **Forgetting the d | c check**: Many solutions claimed when impossible.
- **Negative solutions allowed**: Diophantine usually allows negative x,y unless specified nonnegative (Frobenius is nonnegative).
- **Unique solution**: There are infinitely many if any, parametrized by t.
- **d= gcd(a,b) in the steps**: Always divide the coefficients by d when writing the general solution.
