# Fermat's Little Theorem

Fermat's Little Theorem is one of the most useful and beautiful results in elementary number theory. It provides a powerful necessary condition for primality and a practical way to reduce exponents when working modulo a prime.

Stated simply: if p is prime and a is not divisible by p, then a^{p-1} ≡ 1 mod p. Equivalently a^p ≡ a mod p (this form holds even if p divides a).

The theorem is the foundation for the Fermat primality test (probabilistic), underlies RSA and Diffie-Hellman exponentiation, and is a special case of Euler's theorem. Its proof can be given via group theory (order of multiplicative group) or combinatorially (necklaces) or by induction using binomial theorem.

## Statement in Two Forms

<!-- section: fermat-statement -->

**Fermat's Little Theorem (first form)**: If p is a prime and gcd(a, p) = 1, then
\[
a^{p-1} \equiv 1 \pmod{p}.
\]

**Second form** (sometimes more convenient):
\[
a^p \equiv a \pmod{p}
\]
for any integer a (no coprimeness needed).

The second follows from the first when p ∤ a by multiplying both sides by a; when p | a both sides are 0 mod p.

**ELI5**

- Raise any number not multiple of p to the (p-1) power and you get 1 more than a multiple of p. It's like the numbers "cycle" with period dividing p-1 in the multiplicative group mod p.
- The a^p ≡ a form is a cute way to remember that p "eats" the exponent difference.

**Worked Example:**

**Verify for p=5, a=3**

3^4 =81. 81 /5 =16*5=80, remainder 1. 81≡1 mod5 yes.

3^5=243. 243-48*5=243-240=3, so 3^5≡3 mod5.

## Group-Theoretic Proof (Lagrange)

<!-- section: group-proof -->

The multiplicative group (\mathbb{Z}/p\mathbb{Z})^\times of nonzero residues mod p has exactly p-1 elements and is a group under multiplication mod p.

By Lagrange's theorem, the order of any element a divides the group order p-1.

Thus a^{p-1} = (a^{ord})^{k} = e^k =1 in the group, i.e. a^{p-1} ≡1 mod p.

This is the shortest modern proof once one knows basic group theory.

A purely elementary proof uses that the numbers a,2a,...,(p-1)a are all distinct nonzero mod p (since p prime, a invertible), hence they are a permutation of 1..p-1. Their products are equal: a^{p-1} (p-1)! ≡ (p-1)! mod p, and since (p-1)! not 0 mod p (Wilson or just no factor p), cancel to get a^{p-1}≡1.

**ELI5**

- The nonzero numbers mod p form a "club" of p-1 members closed under multiplication (with inverses). Raising a member to the power of the club size always brings it back to the "identity" 1.
- The permutation proof: multiplying the whole set by a just shuffles it, so the product of the set is multiplied by a^{p-1}, but shuffling doesn't change the product, hence a^{p-1}≡1.

**Worked Example:**

**Using the permutation argument for p=7, a=3**

The set {1,2,3,4,5,6} multiplied by 3 mod7: 3,6,2,5,1,4 (i.e. 3*1=3,3*2=6,3*3=9≡2,3*4=12≡5,3*5=15≡1,3*6=18≡4).

The set is just reordered. Product 1*2*3*4*5*6 = 720.

Left: 3^6 * same product. So 3^6 ≡1 mod7. Indeed 729=104*7 +1? 7*104=728, yes 729-728=1.

## Fermat Primality Test and Pseudoprimes

<!-- section: fermat-test -->

If p is prime then for random a coprime to p we have a^{p-1}≡1 mod p.

Conversely, if for a chosen a we have a^{p-1} ≢1 mod n (and gcd(a,n)=1), then n is definitely composite.

This is the Fermat test. If it passes for several bases we say n is a probable prime.

There exist composite numbers that pass the test for all a coprime to them: Carmichael numbers (square-free composites n where for every prime p|n, p-1 divides n-1).

The smallest is 561=3*11*17.

**ELI5**

- If a number fails the "a^{n-1}≡1" test for even one a, it cannot be prime. Passing many tests makes it "probably prime" for practical purposes (used in real crypto libraries with care).
- Carmichael numbers are "liars" that pretend to be prime for the Fermat test; they are rare but must be guarded against with stronger tests (Miller-Rabin).

**Worked Example:**

**561 as Carmichael**

For any a coprime to 561, a^{560} ≡1 mod561. It passes Fermat test for all such a, yet 561=3*11*17 is composite. (Because 2|560,10|560,16|560.)

## Applications to Exponentiation

<!-- section: fermat-exponentiation -->

When computing a^k mod p with p prime, reduce the exponent k modulo (p-1): a^k ≡ a^{k mod (p-1)} mod p (when p∤a).

This makes enormous exponents feasible.

Combined with repeated squaring (binary exponentiation), even 2048-bit exponents mod 2048-bit prime are fast.

In RSA, the private exponent d satisfies e d ≡1 mod φ(n), and by Euler/Fermat the powers cycle with that period.

**ELI5**

- The exponents live on their own clock of size p-1. You only need to know the time on that smaller clock to know what power you get.
- Binary exponentiation is like using the binary representation of the (reduced) exponent to square and multiply only log times.

**Worked Example:**

**Reduce 3^{1000} mod 7**

p-1=6. 1000 mod 6: 1000/6=166*6=996, remainder 4.

So 3^{1000} ≡ 3^4 =81 ≡4 mod7 (as 7*11=77,81-77=4).

## Common Mistakes

- **Applying without the coprimeness check**: If p divides a then a^{p-1} ≢1 in general (both sides 0 if using a^p form wrongly). Always verify gcd(a,p)=1 for the a^{p-1} form.
- **Using for composite n as a primality proof**: Passing Fermat test for a few bases does not prove primality (Carmichael counterexamples). Use deterministic Miller-Rabin or full factorization for proof.
- **Forgetting to reduce exponent correctly**: The reduction a^k = a^{q(p-1) + r} = (a^{p-1})^q * a^r ≡ 1^q * a^r only holds when gcd(a,p)=1.
- **Confusing with Euler's theorem**: Fermat is the special case of Euler when m=p prime (φ(p)=p-1). Euler works for composite moduli too but with φ(n) which is harder to compute.
- **p=2 special case**: For p=2, a odd ⇒ a^1 ≡1 mod2, which holds.
