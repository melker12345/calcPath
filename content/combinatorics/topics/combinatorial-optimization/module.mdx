# Combinatorial Optimization

Matroids, matchings, network flows, and linear programming for discrete optimization problems.

A matroid is a combinatorial structure that abstracts the notion of linear independence. Matroids appear in linear algebra, graph theory, combinatorial optimization, and many other areas. Their theory provides a unified min-max and algorithmic framework for a wide range of "independence" problems.

## Definition via Independent Sets (and Other Axioms)

<!-- section: definition-axioms -->

A matroid M = (E, I) consists of a finite ground set E and a family I of subsets of E (the independent sets) satisfying:

1. ∅ ∈ I.
2. If A ∈ I and B ⊂ A then B ∈ I (hereditary / downward closed).
3. (Augmentation) If A, B ∈ I and |A| < |B|, then there exists e ∈ B \ A such that A ∪ {e} ∈ I.

Equivalent axiomatizations exist using bases, circuits (minimal dependent sets), the rank function (submodular, monotone), and the closure operator.

**ELI5**

- Independent sets behave exactly like linearly independent vectors: you can always extend a smaller one using an element from a larger one.
- The axioms are chosen so that all the good theorems about bases, rank, etc. continue to hold.

**Worked Example:**

Take E = {1,2,3}, I = all subsets of size ≤ 2 except {1,2}. Check axioms: augmentation may fail between {1,3} and {2} or adjust; this may not be a matroid. A valid graphic example below.

## Important Examples

<!-- section: examples -->

- **Vector matroid / linear matroid**: columns of a matrix; independent = linearly independent columns.
- **Graphic matroid**: edges of a graph; independent = acyclic sets of edges (forests). Rank of a set of edges = v − c in the subgraph.
- **Partition matroid / transversal**: given a partition of E into blocks with capacities, independent sets take at most the capacity from each block.
- **Uniform matroid U_{k,n}**: all subsets of size ≤ k are independent.
- **Fano matroid**, projective geometries, etc.

Many theorems true for vector spaces or for graphs have matroid generalizations.

**ELI5**

- In a graph, a forest is "independent" because it has no cycles (the circuit axiom: a cycle is a minimal dependent set).
- In a vector space the circuits are the minimal linearly dependent sets (relations).

**Worked Example:**

Cycle graph C_4 has graphic matroid with 4 elements (edges). A basis (max forest) has 3 edges (a spanning tree of the vertices? C_4 has 4 verts, rank 3). Any 3 edges form a path (forest); all 4 form a circuit.

## Optimization and Min-Max Theorems

<!-- section: optimization -->

The greedy algorithm (sort by weight descending, add if remains independent) finds a maximum-weight basis exactly when the set system is a matroid.

The matroid intersection theorem gives a min-max formula for the largest set independent in *two* different matroids on the same ground set; it is solved in polynomial time.

These results generalize Kruskal's MST algorithm, bipartite matching, and many other classical optimization results.

**ELI5**

- Greedy works perfectly for matroids: always pick the next best item that doesn't create a "circuit".
- For two matroids (e.g. one graphic, one partition) the common independent sets (e.g. matchings that are also forests) have a clean min-max size.

**Worked Example:**

Kruskal's algorithm for MST is exactly the matroid greedy algorithm on the graphic matroid with edge weights.

## Common Mistakes

- **Thinking every hereditary family is a matroid**: The augmentation axiom is crucial; without it greedy can fail.
- **Confusing circuits with bases**: Circuits are the minimal *dependent* sets; bases are the maximal *independent* sets.
- **Assuming all matroids are representable over a field**: Many are not (e.g. the non-Pappus matroid); the abstract theory is richer.
- **Forgetting that graphic matroids are a proper subclass**: Not every matroid comes from a graph.
- **Overlooking the algorithmic power**: Matroid algorithms (greedy, intersection, union, etc.) give polynomial-time solutions for many problems that look hard when stated in the original language.

A matching is a set of edges that share no vertices. Matchings are among the most fundamental objects in graph theory and combinatorial optimization; they model pairings, assignments, and resource allocation.

The theory is especially rich in bipartite graphs, where min-max theorems and efficient algorithms abound.

## Definitions and Basic Facts

<!-- section: definition-matching -->

A matching M is a subset of edges with no two sharing a vertex.

A vertex is matched (covered) if incident to an edge in M; otherwise free (exposed).

A maximum matching has the largest possible number of edges.

An augmenting path with respect to M is an alternating path (edges alternately in and out of M) that begins and ends at free vertices.

By Berge's lemma, a matching is maximum if and only if it admits no augmenting path.

**ELI5**

- A matching is a collection of "paired" edges that don't fight over people (vertices).
- An augmenting path is an alternating chain that lets you flip and get one more pair.

**Worked Example:**

In a path of three edges a-b-c-d-e, the matching {a-b, c-d} has size 2; there is an augmenting path e-d-c-b-a? Wait edges. Standard P_4 (4 verts) max matching size 2.

## Hall's Theorem in Bipartite Graphs

<!-- section: halls-theorem -->

Hall's marriage theorem: Let G be bipartite with parts L and R. There is a matching that saturates (covers) every vertex of L if and only if for every S ⊆ L we have |N(S)| ≥ |S|, where N(S) is the set of neighbors of S in R.

This is a necessary and sufficient condition and a prototype min-max theorem.

**ELI5**

- To marry off every boy (L) to a girl (R) such that each girl gets at most one, you need that any group of k boys collectively knows at least k girls.
- If some group of boys knows fewer girls than boys, obviously impossible.

**Worked Example:**

Boys A,B,C; girls 1,2,3. A knows 1,2; B knows 1; C knows 1,2,3. For S={A,B} N(S)={1,2} size 2 ≥2. Hall holds; a perfect matching from L exists (B-1, A-2, C-3).

## Algorithms and Min-Max

<!-- section: algorithms -->

In bipartite graphs: reduce to max flow (unit capacities from source to L, edges L-R capacity 1, R to sink). Ford-Fulkerson or Dinic gives max matching. Hopcroft-Karp is faster O(E√V).

In general graphs: Edmonds' blossom algorithm contracts odd alternating cycles ("blossoms") and recurses; polynomial time.

König's theorem (bipartite): size of max matching = size of min vertex cover.

**ELI5**

- Flow models "sending one unit of assignment along an edge without exceeding capacities."
- Blossoms are the odd-cycle obstacles that require special shrinking in the non-bipartite case.

**Worked Example:**

Kruskal or any max cardinality matching algorithm on a small graph will find the largest set of disjoint edges.

## Common Mistakes

- **Allowing shared vertices in a 'matching'**: By definition no two edges in a matching share a vertex.
- **Confusing matching with edge cover or vertex cover**: A vertex cover touches *all* edges; a matching is a *packing* of edges.
- **Thinking Hall's condition is only sufficient**: It is necessary and sufficient.
- **Forgetting that in non-bipartite graphs min vertex cover can be strictly larger than max matching** (e.g., a triangle: matching 1, vertex cover 2).
- **Assuming perfect matchings always exist when |L|=|R|**: Hall's (or Tutte's) condition may fail.

Network flows model the movement of a commodity through a network with capacities on the arcs. The maximum flow problem and its min-cut dual are among the most important and versatile tools in combinatorial optimization.

## Flow Networks and Feasible Flows

<!-- section: flow-networks -->

A flow network G = (V,E) has distinguished source s and sink t, and a capacity function c : E → ℝ≥0.

A flow f : E → ℝ satisfies:

- Capacity: 0 ≤ f(e) ≤ c(e) for every edge.
- Conservation: for every v ≠ s,t, sum_{edges into v} f = sum_{edges out of v} f.

The value |f| of the flow is the net flow out of s.

**ELI5**

- Think of pipes with limited throughput (capacity) carrying water from s to t.
- At every junction (non s/t) the amount flowing in must equal the amount flowing out.

**Worked Example:**

s connected to a (cap 5) and b (cap 3); a to t (4), b to t (4), a to b (1). A feasible flow can send 4 from a to t, 3 from b to t (total 7); the a-b edge can carry extra if needed but is not required for max.

## The Max-Flow Min-Cut Theorem

<!-- section: maxflow-mincut -->

The max-flow min-cut theorem asserts that the maximum possible flow value equals the minimum capacity of an s-t cut.

An s-t cut is a partition of V into S ∋ s and T = V\S ∋ t; its capacity is the sum of c(e) over edges e from S to T.

**ELI5**

- You cannot push more flow across the network than the total "pipe capacity" crossing any cut that separates s from t.
- The theorem says there is always a bottleneck cut whose capacity exactly matches the max flow you can achieve.

## Algorithms and Applications

<!-- section: applications -->

Ford-Fulkerson / Edmonds-Karp: repeatedly augment along a path in the residual graph (forward residual capacity or backward flow) until no s-t path in residual.

With integer capacities the max flow is integer and the method terminates.

Applications: bipartite matching, assignment problem, transportation, circulation with demands, image segmentation (min-cut), project selection, etc.

**ELI5**

- Residual graph lets you "undo" previous flow decisions to reroute.
- Many real-world "shipping with capacities" or "matching with constraints" reduce to a max-flow computation.

**Worked Example:**

To compute max bipartite matching, create flow network with s to all left (cap 1), left to right according to edges (cap 1), right to t (cap 1). Max flow value = size of max matching.

## Common Mistakes

- **Allowing negative flows or violating conservation**: Feasibility requires both.
- **Confusing the residual graph with the original**: Backward edges have capacity equal to current flow (can cancel).
- **Thinking max flow is always the sum of capacities out of s**: That is only an upper bound; other cuts may be tighter.
- **Forgetting that the min cut found from residual reachability is a proof of optimality**: After running any max-flow alg, the reachable set from s in residual gives a cut of capacity equal to the flow value.
- **Assuming all flows are integral**: With fractional capacities you may need fractional flows; the integral flow theorem holds when all capacities are integers.

Linear programming (LP) is the problem of optimizing a linear objective over a polyhedron defined by linear inequalities. It is both a fundamental algorithmic tool and the source of powerful min-max theorems and duality in combinatorics.

Many combinatorial optimization problems admit natural LP formulations that are integral, so that LP solvers (or combinatorial algorithms derived from them) solve the discrete problem exactly.

## The Linear Programming Problem

<!-- section: lp-problem -->

In inequality form: maximize c^T x subject to A x ≤ b, x ≥ 0.

(Equality form obtained by adding slack variables.)

The feasible region is a convex polyhedron. If the optimum is finite it is attained at a vertex (a basic feasible solution with at most m tight constraints in m-dimensional space).

**ELI5**

- You have a linear "profit" function c·x and a bunch of linear resource limits A x ≤ b.
- The best point is always at a "corner" of the feasible region.

**Worked Example:**

Max 3x + 4y s.t. x + y ≤ 5, x≥0, y≥0.

Vertices: (0,0) value 0; (5,0) value 15; (0,5) value 20.

Optimum 20 at (0,5).

## Duality

<!-- section: duality -->

Every primal LP has a dual LP. For the standard max c x s.t. A x ≤ b, x≥0 the dual is min b y s.t. A^T y ≥ c, y ≥0.

Weak duality: any feasible primal x and dual y satisfy c x ≤ b y.

Strong duality: if both problems are feasible and bounded, then the optimal values are equal, and optimal solutions exist.

**ELI5**

- The dual provides a "pricing" or "certificate" problem whose value upper-bounds the primal.
- At optimality the best primal profit exactly equals the best dual "cost".

**Worked Example:**

Primal max x1 + x2 s.t. x1 + 2 x2 ≤ 4, x≥0.

Dual min 4 y s.t. y ≥1 (x1 coeff), 2y ≥1 (x2), y≥0.

Optimal primal at (4,0) value 4; dual at y=1 value 4. Equal.

## Combinatorial Applications

<!-- section: combinatorial-apps -->

- Bipartite matching / assignment as LP with totally unimodular (TU) constraint matrix → integral optima.
- Network flows and min-cost flows.
- Shortest paths, min spanning trees (via LP or duals).
- Many packing and covering problems admit LP relaxations whose integrality gap is 1 or can be rounded.

When the matrix is TU (network matrices, interval matrices, bipartite incidence, etc.) every vertex of the polyhedron is integer, so LP solves the IP exactly.

**ELI5**

- You write the combinatorial problem as "maximize profit subject to linear packing constraints"; if the matrix is nice the best fractional solution is already integral.
- Duality often recovers the classical min-max theorems (König, max-flow min-cut, etc.).

**Worked Example:**

The bipartite matching LP (variables x_e for edges, degree constraints =1 or ≤1, max sum x) has integral vertices (permutation matrices or 0-1 matchings) because the constraint matrix is TU. Hence the LP value equals the integer matching size.

## Common Mistakes

- **Thinking every LP has an integral optimum**: Only when the polyhedron has integer vertices (TU or other conditions).
- **Forgetting strong duality requires feasibility of both sides**: If one is infeasible the other may be unbounded.
- **Confusing simplex (vertex walking) with interior-point methods**: Both solve LP; simplex has combinatorial flavor.
- **Assuming all combinatorial problems have small integrality gap**: Some have large gaps; approximation algorithms study the gap.
- **Neglecting degeneracy and multiple optima**: A vertex may have several bases; complementary slackness helps certify optimality.
