# Exponential generating function

Exponential generating functions (EGFs) are the natural tool when the objects being counted are *labeled* — distinct people, numbered positions, or distinguishable atoms.

The division by n! in the definition cancels the n! ways to label an n-element structure, yielding clean algebraic expressions for sets, sequences, and cycles of labeled components.

## Definition of EGFs

<!-- section: definition-egf -->

The exponential generating function for the sequence a_n (number of structures on an n-element labeled set) is

$$A(x) = \sum_{n \ge 0} \frac{a_n}{n!} x^n.$$

To recover the count: a_n = n! [x^n] A(x).

**ELI5**

- Because labels can be permuted in n! ways, we "pre-divide" by n! so that the series coefficients directly reflect the "shape" count.
- For permutations of [n] there are n! of them; their EGF simplifies to sum x^n = 1/(1-x) — beautiful.

**Worked Example:**

Number of ways to choose a subset of [n] and then order it: a_n = n! (any permutation of any subset? Wait no.

For all permutations: a_n = n!, EGF = sum n! (x^n / n!) = sum x^n = 1/(1-x).

## Labeled Structures and Set Constructions

<!-- section: labeled-structures -->

For labeled objects the basic building blocks translate to:

- A single labeled atom: x
- A non-empty set of labeled atoms: e^x − 1
- A sequence of labeled structures: 1 / (1 − A(x))
- A set of labeled structures: exp(A(x))

These are far cleaner than the corresponding ordinary generating functions for the same labeled counts.

**ELI5**

- "A set of cycles" (permutations) has EGF exp( − ln(1−x) ) = 1/(1−x).
- "Set partitions" : the blocks are non-empty sets, so EGF exp( e^x − 1 ).

**Worked Example:**

Number of set partitions of an n-set (Bell number B_n).

The EGF is exp(e^x − 1).

B_4 = 15. Indeed [x^4]/4! of the series gives the count after *4!.

## Relation to Ordinary Generating Functions

<!-- section: relation-to-ogf -->

When structures are unlabeled, OGFs are preferred. When labels matter (or when we count permutations, functions, mappings), EGFs usually produce simpler closed forms and composition rules that mirror the combinatorial constructions directly.

Many species in combinatorial enumeration theory are defined via their EGFs.

**ELI5**

- If the people are distinguishable, EGFs "factor out" the labeling so the algebra sees only the grouping.
- Converting between OGF and EGF is possible via Borel or other transforms but usually we choose the right one from the start.

**Worked Example:**

Derangements (permutations with no fixed points): d(n) ≈ n!/e.

EGF: e^{−x} / (1 − x).

Then d(n) = n! [x^n] (e^{-x}/(1-x)).

For n=4: 9.

## Common Mistakes

- **Forgetting the n! multiplier when extracting counts from EGF**: a_n = n! [x^n] A(x), not just the coefficient.
- **Using EGF for unlabeled objects like integer partitions**: The /n! would be meaningless or wrong; use OGF instead.
- **Confusing the constructions**: "set of" is exp, "sequence of" is 1/(1-A); swapping them is a common error.
- **Treating EGFs as probability generating functions**: They are formal counting tools; the /n! is combinatorial, not probabilistic.
- **Ignoring the empty structure**: The constant term is almost always 1 for the empty labeled set.
