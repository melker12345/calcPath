# Linear programming

Linear programming (LP) is the problem of optimizing a linear objective over a polyhedron defined by linear inequalities. It is both a fundamental algorithmic tool and the source of powerful min-max theorems and duality in combinatorics.

Many combinatorial optimization problems admit natural LP formulations that are integral, so that LP solvers (or combinatorial algorithms derived from them) solve the discrete problem exactly.

## The Linear Programming Problem

<!-- section: lp-problem -->

In inequality form: maximize c^T x subject to A x ≤ b, x ≥ 0.

(Equality form obtained by adding slack variables.)

The feasible region is a convex polyhedron. If the optimum is finite it is attained at a vertex (a basic feasible solution with at most m tight constraints in m-dimensional space).

**ELI5**

- You have a linear "profit" function c·x and a bunch of linear resource limits A x ≤ b.
- The best point is always at a "corner" of the feasible region.

**Worked Example:**

Max 3x + 4y s.t. x + y ≤ 5, x≥0, y≥0.

Vertices: (0,0) value 0; (5,0) value 15; (0,5) value 20.

Optimum 20 at (0,5).

## Duality

<!-- section: duality -->

Every primal LP has a dual LP. For the standard max c x s.t. A x ≤ b, x≥0 the dual is min b y s.t. A^T y ≥ c, y ≥0.

Weak duality: any feasible primal x and dual y satisfy c x ≤ b y.

Strong duality: if both problems are feasible and bounded, then the optimal values are equal, and optimal solutions exist.

**ELI5**

- The dual provides a "pricing" or "certificate" problem whose value upper-bounds the primal.
- At optimality the best primal profit exactly equals the best dual "cost".

**Worked Example:**

Primal max x1 + x2 s.t. x1 + 2 x2 ≤ 4, x≥0.

Dual min 4 y s.t. y ≥1 (x1 coeff), 2y ≥1 (x2), y≥0.

Optimal primal at (4,0) value 4; dual at y=1 value 4. Equal.

## Combinatorial Applications

<!-- section: combinatorial-apps -->

- Bipartite matching / assignment as LP with totally unimodular (TU) constraint matrix → integral optima.
- Network flows and min-cost flows.
- Shortest paths, min spanning trees (via LP or duals).
- Many packing and covering problems admit LP relaxations whose integrality gap is 1 or can be rounded.

When the matrix is TU (network matrices, interval matrices, bipartite incidence, etc.) every vertex of the polyhedron is integer, so LP solves the IP exactly.

**ELI5**

- You write the combinatorial problem as "maximize profit subject to linear packing constraints"; if the matrix is nice the best fractional solution is already integral.
- Duality often recovers the classical min-max theorems (König, max-flow min-cut, etc.).

**Worked Example:**

The bipartite matching LP (variables x_e for edges, degree constraints =1 or ≤1, max sum x) has integral vertices (permutation matrices or 0-1 matchings) because the constraint matrix is TU. Hence the LP value equals the integer matching size.

## Common Mistakes

- **Thinking every LP has an integral optimum**: Only when the polyhedron has integer vertices (TU or other conditions).
- **Forgetting strong duality requires feasibility of both sides**: If one is infeasible the other may be unbounded.
- **Confusing simplex (vertex walking) with interior-point methods**: Both solve LP; simplex has combinatorial flavor.
- **Assuming all combinatorial problems have small integrality gap**: Some have large gaps; approximation algorithms study the gap.
- **Neglecting degeneracy and multiple optima**: A vertex may have several bases; complementary slackness helps certify optimality.
