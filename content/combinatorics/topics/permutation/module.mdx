## Permutations with Repetition

<!-- section: permutations-with-repetition -->

When repetition is allowed, each of the k positions independently has n choices, giving n^k total strings.

This is simply the product rule applied k times. If the "objects" are chosen from a larger set with replacement, the formula changes from the no-repetition case.

**ELI5**

- With repetition you can have passwords like "AAA" or "1212". Each slot doesn't care what was chosen before.
- Without repetition it is like dealing cards: once a card is used it is gone.

**Worked Example:**

Number of 4-digit PINs (digits 0-9, repetition allowed, including 0000): 10^4 = 10000.

If leading zeros disallowed for a 4-digit number: first digit 1-9 (9 options), remaining 10 each: 9 Ã— 10^3 = 9000.

- The word "MISSISSIPPI" has many identical letters, so we overcount by swapping the I's among themselves unless we divide by 4! for the I's (and similarly for S's and P's).

## Common Mistakes

- **Confusing P(n,k) with \binom{n}{k}**: P(n,k) keeps the order; \binom{n}{k} does not. Multiply a combination by k! to turn it into the corresponding permutations.
- **Forgetting to divide by repeats in multiset case**: "BOOK" is not 4!; identical O's require /2!.
- **Using n! for circular without fixing one**: This overcounts by exactly n (the rotations).
- **Thinking 0! = 0**: It is 1; this is crucial for the formula P(n,0) = 1 and empty arrangements.
- **Ignoring that repetition changes the model entirely**: Always check the problem statement for "with replacement" or "repetition allowed".
