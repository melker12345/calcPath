# Generating function

A generating function packages an entire infinite sequence into a single formal power series. By manipulating the series algebraically we can often extract the nth coefficient — i.e., solve the original counting problem — without listing terms.

Ordinary generating functions (OGFs) are the workhorse for unlabeled or unordered structures; they turn addition of sizes into multiplication of series.

## Ordinary Generating Functions

<!-- section: ordinary-generating -->

For a sequence a_0, a_1, a_2, … the ordinary generating function is

$$A(x) = \sum_{n=0}^\infty a_n x^n = a_0 + a_1 x + a_2 x^2 + \cdots$$

The exponent marks the parameter we care about (usually size n); the coefficient a_n is the answer to the counting question for that n.

**ELI5**

- Imagine a vending machine that for each possible "size" n drops out a_n different candy combinations. The generating function is the "master list" written as a polynomial you can add and multiply.
- The beauty is that algebraic operations on the master list correspond to combining choices.

**Worked Example:**

Number of ways to make n cents using only 1-cent coins: a_n = 1 for all n.

GF: 1 + x + x^2 + x^3 + … = 1/(1 − x).

## Operations and Their Combinatorial Meaning

<!-- section: operations-on-gfs -->

- Sum A(x) + B(x): disjoint union of two classes.
- Product A(x)B(x): ordered pairs (or sequences of two independent structures) whose sizes add.
- 1/(1 − A(x)): any number (including zero) of copies of structures from A, sizes adding (compositions).

These turn recursive or compositional counting into algebra.

**ELI5**

- Multiplying two GFs is like "first choose a red item of size k (a_k ways) and independently a blue item of size n-k (b ways)"; the coeff sums all splits.
- The geometric series 1/(1-x) = sum x^n corresponds to "any number of 1-cent coins."

**Worked Example:**

Unrestricted number of identical apples and identical oranges.

GF per fruit type: 1 + x + x^2 + … = 1/(1-x).

Combined: [1/(1-x)]^2 = sum (n+1) x^n.

Thus there are n+1 ways to have total n fruits (0 to n apples, rest oranges).

## Solving Classic Counting Problems

<!-- section: solving-counting-problems -->

Many classic problems become routine once the GF is written:

- Integer partitions → product over part sizes 1/(1-x^k)
- Compositions with restricted part sizes → 1 / (1 − (x + x^2 + … + x^m))
- Tilings, paths, matchings → often lead to linear recurrences whose characteristic equation appears in the denominator.

After writing the GF one can expand, use partial fractions, or extract [x^n] via recurrence or other tools.

**ELI5**

- "How many ways to pay exactly n cents with standard US coins?" Write the product of geometric series for each denomination; the coefficient of x^n is the answer.
- The algebra does the heavy casework for you.

**Worked Example:**

Number of ways to tile a 2 × n board with dominoes (each covers two squares).

Let f_n be the number. A tiling either ends with a vertical domino (then f_{n-1} before) or two horizontal (f_{n-2} before).

Thus f_n = f_{n-1} + f_{n-2}, f_0=1, f_1=1 → Fibonacci.

The GF F(x) = 1 / (1 − x − x^2) encodes all f_n.

## Common Mistakes

- **Confusing ordinary with exponential generating functions**: OGFs for unlabeled/unordered; EGFs (with /n!) for labeled or ordered structures like permutations.
- **Treating the series as convergent real functions too early**: In combinatorics we usually work formally; analytic questions come later.
- **Forgetting the constant term**: The empty structure almost always contributes the 1 (x^0 term).
- **Misidentifying what the exponent represents**: Make sure x marks exactly the parameter you want to count (size, weight, etc.).
- **Writing the wrong factor for "unlimited supply"**: Unlimited identical items of one type → 1/(1-x^k) not (1+x^k).
